## 一、总览：P0-P2 三级优先级

ExchangeAlpha PUSH 采用 P0-P2 三级优先级体系，新老用户均适用此优先级框架，P0/P1 可突破频控和免打扰，P2 严格受限。

│                    ExchangeAlpha PUSH 优先级体系                         │

├──────────┼──────────┼──────────────┼──────────┼────────────┤
│ P0 关键   │ 突破所有   │  ✅ 可突破    │ 安全告警   │  新老一致   │

├──────────┼──────────┼──────────────┼──────────┼────────────┤

## 二、全局频控规则（新老用户统一遵守）

| 维度 | 规则 | 说明 |
|------|------|------|
| **周硬上限** | **≤ 5 条 / 用户 / 周** | 跨 Push + 邮件 + 短信累计，最高优先级铁律 |
| **日上限** | ≤ 2 条 / 用户 / 天（P2 活动类 ≤2，P2 推荐类 ≤1） | 避免单日骚扰 |
| **月上限** | ≤ 20 条 / 用户 / 月 | 长期频控兜底 |
| **跨渠道合并** | 默认开启，Push + 邮件 + 短信全局累计 | 避免多通道同时轰炸 |
| **免打扰时段** | **21:00 – 09:00 静默** | P0 始终送达，P1 可突破，P2 延迟至 09:00 |
| **用户偏好过滤** | 发送前校验分类开关 | 用户关闭某分类则跳过（P0 除外） |

## 三、新用户 PUSH 策略（注册 0-30 天）

### 3.1 Onboarding 期（Day 0-7）

| 参数 | 设定 |
|------|------|
| **频控** | **每天 ≤ 1 条** |
| **总条数** | 7 天内最多 7 条 |
| **推送优先级** | P2（互动），受频控限制 |
| **推送类型** | 活动推送 / 任务提醒 / 个性化推荐 |

### 3.2 激活期（Day 7-30）

| 参数 | 设定 |
|------|------|
| **频控** | **每周 2-4 条** |
| **推送优先级** | P2（互动）为主，P1（交易提醒）为辅 |
| **推送类型** | 交易提醒（价格预警、订单成交）+ 个性化推荐 + 活动推送 |
| **策略重点** | 以**事件触发**替代定时广播（事件触发转化率是定时广播的 2 倍） |

**内容策略**：
- 交易提醒：用户关注的币种价格突破关键点位时实时推送（P1）
- 市场异动：BTC/ETH 涨跌幅超 5%，推送行情简报
- 产品推荐：根据用户浏览行为推荐跟单/网格/理财（P2-个性化推荐）
- 活动推送：限时活动提醒（P2-活动推送，每周 ≤2 条）

| **策略重点** | 个性化推荐 + 周报汇总，提升 LTV |

| 用户分层 | 推送内容 | 频率 | 举例 |

## 五、沉默 & 流失用户 PUSH 策略

### 5.1 沉默用户（7-30 天未活跃）

| 参数 | 设定 |
|------|------|
| **频控** | **7-14 天 1 条** |
| **推送优先级** | P2（召回推送） |

**渐进式触达策略**：

                DeepLink: deeplink://home

                DeepLink: deeplink://coupon_center

                DeepLink: deeplink://deposit

### 5.2 流失用户（30 天+ 未活跃）

| 参数 | 设定 |
|------|------|
| **频控** | **总量 ≤ 8-10 条后停止**——超过 10 条无响应即永久终止触达 |
| **推送优先级** | P2（召回推送） |

**防骚扰机制**：累计推送超过 10 条且用户无任何响应行为（未打开/未登录），系统自动将该用户移出召回池，避免持续骚扰导致卸载。

## 六、推送内容规范

### 6.1 格式规范

| 字段 | 规范 | 示例 |
|------|------|------|
| **推送标题** | ≤ 30 字符，吸引眼球 | `"BTC 突破 10 万 U！"` |
| **推送正文** | ≤ 80 字符，含行动号召 | `"立即参与瓜分 50,000 USDT 奖励 →"` |
| **跳转链接** | 必须配置 DeepLink | `deeplink://contract_copy_trading` |
| **关联活动 ID** | 必须填写，便于归因 | `ACT_2025_0501` |

### 6.2 禁止内容

- ❌ 绝对化用语："最高"、"第一"、"保证"
- ❌ 过度营销："立即充值 10000U"
- ❌ 未经法务审核的合规文案
- ❌ 虚假紧迫感："仅剩 1 份"

### 6.3 变量替换

支持动态变量，运行时替换为实际值：
- `{username}` → 用户昵称
- `{amount}` → 奖励金额
- `{prize_name}` → 奖品名称
- `{coin}` → 币种名称

## 七、用户推送偏好（降低退订率）

替代传统二元开关（开/关），提供 **4 类推送分类开关**：

| 分类 | 默认 | 说明 |
|------|------|------|
| **交易提醒** | 开启 | 价格预警、订单成交、止盈止损 |
| **活动通知** | 开启 | 新活动、交易大赛、空投福利 |
| **市场资讯** | 开启 | 行情异动、周报汇总、市场分析 |
| **安全通知（P0）** | 强制开启，不可关闭 | 账户安全、风控通知、支付失败 |

**效果**：分类开关比二元开关降低退订率约 **30-50%**。

## 八、频控执行顺序

```
P0/P1 优先级判断
       ↓
推送类型独立频控（P2-活动 ≤2条/日，P2-推荐 ≤1条/日）
       ↓
跨渠道合并频控（Push + 邮件 + 短信累计）
       ↓
免打扰时段检查（21:00-09:00 → P2 延迟至 09:00）
       ↓
用户偏好过滤（用户关闭该分类 → 跳过，P0 除外）
       ↓
全局周硬上限（≤5条/周）
       ↓
✅ 发送
```
