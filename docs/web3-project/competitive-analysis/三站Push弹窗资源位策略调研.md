# ExchangeGamma、ExchangeEpsilon、ExchangeZeta 三站 PUSH / 首页弹窗 / 站内资源位策略调研

> 调研日期：2026-06-08

## 一、ExchangeEpsilon（ExchangeEpsilon）

### 1.1 推送通知体系

ExchangeEpsilon的通知体系较为成熟和克制，支持用户高度自定义：

| 推送类型 | 内容 | 策略特点 |
|---------|------|---------|
| **交易通知** | 订单成交、止损止盈触发 | 毫秒级实时推送，核心交易体验 |
| **行情提醒** | 价格预警、资金费率异动 | 用户自设阈值，主动触发 |
| **活动通知** | Launchpool、Earn 新品、交易大赛 | 频率控制严格，不轰炸 |
| **系统公告** | 合约费率调整、维护升级 | 强制性推送，合规免责 |
| **安全通知** | 登录提醒、提现确认 | 最高优先级，不可关闭 |

通知设置入口：App → 个人中心 → 设置 → 通知管理，支持按类型逐项开关。

### 1.2 首页策略

**2025 年重大改版 — "ExchangeEpsilon UI Refined"：**

- 首页 Pro Mode 支持**可定制 Widget**：市场概览、自选列表、AI 洞察等模块自由拖拽排列
- **AI 驱动个性化**：根据用户持仓/自选推送相关市场分析卡片
- 首页不再用大尺寸弹窗做营销，而是将活动融入**可配置的 Widget 模块**中
- 柔性布局（Flexible Layout）让不同用户看到的首页内容完全不同

### 1.3 弹窗策略

接近**零打扰**策略：

- **首次打开新版本**时有 "What's New" 引导层（半透明遮罩 + 功能亮点标注）
- **大型活动**（如 Megadrop、Launchpool）可能弹出一次全屏 splash，但支持「不再显示」
- 日常运营几乎不用强制弹窗，而是依赖通知中心和 Banner 位
- VIP 用户有专属的 "VIP Portal" 入口，不打扰普通用户

### 1.4 站内资源位矩阵

| 资源位 | 位置 | 用途 | 频次 |
|--------|------|------|------|
| 首页顶部 Banner | 首页最顶部轮播 | 新币上线、Launchpool、重大活动 | 2-4 条轮播 |
| 站内信中心 | 顶部铃铛图标 | 系统公告、活动提醒、奖励过期通知 | 持续可见 |
| Square 广场 | 首页 Tab | UGC 内容 + 官方公告混合 | 常驻 |
| Earn 推荐卡片 | 首页中部 | 理财产品、Staking、Dual Investment | 动态推荐 |
| Widget 模块 | 首页自定义区 | 用户自选内容，非强制 | 用户控制 |

**核心逻辑：用个性化的首页 Widget 替代弹窗轰炸，把选择权还给用户。**

## 二、ExchangeZeta

### 2.1 推送通知体系

ExchangeZeta 的推送策略**比 ExchangeEpsilon 更激进**，且近期重点发力**代理商内容营销 Push**：

| 推送类型 | 内容 | 策略特点 |
|---------|------|---------|
| **交易通知** | 订单、止盈止损 | 标准实时推送 |
| **行情提醒** | 价格异动、资金费率 | 用户自设追踪币种 |
| **代理商内容推送** | 市场分析、交易思路、代币推荐 | **核心差异化功能**：代理商每天可推送 3 次给自己的邀请用户 |
| **活动通知** | Launchpool、交易赛、周年庆 | 频率较高，配合弹窗 |
| **奖励中心提醒** | 奖励到期、未领取红包 | 利用 FOMO 心理召回 |

### 2.2 首页弹窗策略

较**激进**，有明显的获客驱动特征：

- **新用户注册后**：弹窗引导领取 6,200 USDT 迎新礼包，完成 KYC → 充值 → 交易的三步转化
- **大型活动期**（如周年庆、Launchpool 上新）：全屏视觉弹窗，通常配合倒计时或稀缺感
- **奖励过期提醒**：弹窗提示用户奖励即将到期，驱使用户在窗口期内完成操作
- **节日/热点营销**：如《黑神话：悟空》赠送活动（2024.8）、6 周年庆典（2024.9）等借势弹窗

### 2.3 站内资源位矩阵

| 资源位 | 位置 | 用途 | 频次/特点 |
|--------|------|------|----------|
| 首页 Banner | 首页顶部 | Launchpool、交易赛、新币上线 | 高频更新，强视觉冲击 |
| 奖励中心 | 独立模块 | 迎新礼包、任务奖励、每日签到 | **核心转化漏斗入口** |
| 代理商内容展示区 | 首页固定板块 | 代理商发布的市场分析/代币推荐 | 带交易链接，可直接跳转交易 |
| Launchpool 专区 | 首页/活动 Tab | BGB/USDT 锁仓挖矿新币 | 常驻入口 |
| Earn 理财推荐 | 首页 | 活期/定期产品 + 高亮 APY 标签 | 动态推荐 |
| BGB 生态入口 | 个人中心/首页 | BGB 质押、BGB 专属福利 | 推动平台币生态 |

**核心逻辑：弹窗 + Push 双通道高频触达 + 代理商内容营销产生社交裂变 + 奖励中心构建完整转化漏斗。**

## 三、ExchangeGamma（币赢）

### 3.1 推送通知体系

ExchangeGamma 的推送相对中规中矩，偏**社区驱动型**：

| 推送类型 | 内容 | 策略特点 |
|---------|------|---------|
| **新币上线通知** | 新币/代币上线提醒 | 核心推送场景，配合交易赛 |
| **活动推送** | 交易大赛、返佣活动、福利空投 | Push → App 内落地页 |
| **KOL/Creator 内容** | 创作者联盟发布的行情分析 | 通过官方媒体账号转发，非代理商模型 |
| **系统通知** | 维护、升级、安全提醒 | 低频 |

官方在帮助中心单独提供 iOS/Android 的 Push 开启引导教程，说明其将 Push 作为**核心运营渠道**运营。

### 3.2 首页弹窗策略

相对保守：

- **重大活动/新币上线**时触发全屏弹窗（预计频次低于 ExchangeZeta）
- 更多依赖 **KOL 联盟军 + 社区社群（Telegram/Discord/Twitter）** 做二次传播，而非 App 内强弹窗
- 弹窗设计偏活动营销，而非"新人三步转化"强漏斗模型

### 3.3 站内资源位矩阵

| 资源位 | 位置 | 用途 | 特点 |
|--------|------|------|------|
| 首页 Banner | 首页顶部 | 活动/新币/交易赛 | 标准轮播 |
| 活动中心 | 独立入口 | 交易大赛、返佣、福利活动 | 集中展示 |
| Creator Alliance 入口 | 个人中心/首页 | KOL/创作者招募、内容曝光 | ExchangeGamma 特色 |
| GemW 入口 | 首页/导航 | 链上交易（Solana 等） | 2025 年新增 |
| Earn 理财 | 首页/理财 Tab | 活期/定期产品 | 标准配置 |
| WConnect 论坛 | 社区 Tab | 项目上线、Twitter Space | 差异化社交工具 |

**核心逻辑：弱化 App 内强弹窗，强化 KOL/社区裂变传播 + 创作者联盟内容生态 + 高返佣（40%-50%）联盟军模式。**

## 四、三站对比总结

| 维度 | **ExchangeEpsilon** | **ExchangeZeta** | **ExchangeGamma** |
|------|-----------|-----------|----------|
| **Push 策略** | 克制，用户自定义，按类型开关 | 激进，代理商内容推送 3 次/天 | 中规，依赖社区/KOL 二次传播 |
| **首页弹窗** | 极少，首次更新 What's New 引导层 | **频繁**，新人礼包+活动倒计时+奖励过期 | 低频，偏活动驱动 |
| **首页资源位** | 个性化 Widget 模块，用户自选 | 固定 Banner+奖励中心+代理商内容区 | 标准 Banner+活动中心+KOL 入口 |
| **用户分层** | AI 驱动的千人千面首页 | 新老用户漏斗 + 返佣裂变 | 按地区 + KOL 联盟分层 |
| **核心增长引擎** | 产品体验+品牌+AI 个性化 | **弹窗+PUSH 高频触达+代理商裂变** | **KOL 联盟军+社区驱动+高返佣** |
| **弹窗设计哲学** | 少即多，交给用户选择 | 漏斗驱动，每一屏都是转化机会 | 社区优先，App 内保持克制 |

## 五、推送频率与用户生命周期策略

### 5.1 用户生命周期阶段的触达目标

加密货币交易所的推送策略按用户生命周期阶段有明显差异：

| 阶段 | 用户状态 | 核心目标 | 触达频率 | 触达手段 |
|------|---------|---------|---------|---------|
| **新手期** (Day 0-7) | 已注册未 KYC / 未入金 / 未交易 | 推动首次关键行为 | **高频**：每 1-2 天 1 次 | Push + 全屏弹窗 + 站内信 |
| **激活期** (Day 7-30) | 完成首笔交易，尚未形成习惯 | 引导深度使用，扩大交易品类 | **中频**：每周 2-3 次 | Push + Banner + 活动推荐卡片 |
| **成长期** (Day 30-90) | 活跃交易用户 | 提升交易频次和 AUM | **中低频**：每周 1-2 次 | 行情推送 + Earn 推荐 + VIP 升级引导 |
| **成熟期** (90 天+) | 高价值忠实用户 | 留存维护，防止流失 | **低频**：按需触发 | 专属活动 + VIP 权益 + 个性化 Widget |
| **沉睡期** (>14 天未活跃) | 已流失/半流失用户 | 召回激活 | **阶梯式**：14 天/30 天/60 天分段触发 | Push + 邮件 + 奖励弹窗 |
| **流失期** (>90 天未活跃) | 深度流失 | 最后挽救 | **极低频**：60-90 天 1 次 | 大额回归礼包 + 邮件 |

### 5.2 三站推送频率对比（按生命周期）

| 生命周期阶段 | **ExchangeEpsilon** | **ExchangeZeta** | **ExchangeGamma** |
|-------------|-----------|-----------|----------|
| **新手期** (注册 0-7d) | 轻引导：1-2 次站内信 + 可选 KYC 提醒，不强制 Push | **激进**：注册当日弹窗 + 每 1-2 天 Push 1 次引导 KYC/入金/交易，共约 4-6 条/周 | 中规：首次弹窗 + 约 2-3 条 Push（活动+新币），依赖 KOL 拉群转化 |
| **激活期** (7-30d) | 个性化推荐卡片，几乎不用 Push | 活动弹窗 + Push 约 3-4 次/周 + 代理商内容推送 | Push 约 1-2 次/周，主要推交易赛和返佣活动 |
| **成长期** (30-90d) | 行情预警（用户自设）+ Earn 推荐，0 主动 Push | 活动 Push 约 2-3 次/周 + Launchpool 弹窗 + 代理商每日内容推送 | Push 约 1-2 次/周，新币上线通知为主 |
| **成熟期** (90d+) | **极克制**：仅交易/安全通知，靠 Widget 个性化留存 | 专属活动 Push + VIP 邀请 + BGB 生态推送，约 1-2 次/周 | 低频 Push，靠 KOL/创作者联盟社区维持关系 |
| **沉睡期** (14-90d 未活跃) | 回归礼包站内信 + 可选邮件，几乎不用 Push 召回 | **激进召回**：弹窗 + Push 奖励到期提醒 + 回归礼包 + 代理商专属召回 | 依赖社区社群做人工召回，App 内召回较弱 |

### 5.3 Push 频率控制机制对比

| 频率控制维度 | **ExchangeEpsilon** | **ExchangeZeta** | **ExchangeGamma** |
|-------------|-----------|-----------|----------|
| **用户开关粒度** | 按交易/行情/活动/安全/系统 5 类逐一开关 | 按通知类型分类，粒度较粗 | 标准推送开关，无细分 |
| **日推送上限** | 未公开，实际约 0-3 条/天（取决于用户设置） | 普通用户约 3-5 条/天；代理商可额外 Push 3 次/天给直接邀请用户 | 实际约 1-2 条/天 |
| **去重机制** | 活动通知与站内信互斥，不重复推送同一活动 | 弹窗 + Push + 站内信可能叠加触达同一活动（多通道打透） | 较少重复触达 |
| **时段限制** | 无明确限制，但行情类推送 24h 可能触发 | 无明显限制 | 未见明确限制 |
| **退订/免打扰** | 支持全局免打扰 + 按类型静默 | 支持关闭通知 | 支持关闭通知 |

### 5.5 各阶段最佳实践总结

**新手期（0-7 天）— 关键激活窗口：**
- ✅ 注册后 **3 秒内弹窗** 展示核心价值 + 新手福利（用户注意力峰值期）
- ✅ **每 2 天 1 次 Push** 引导未 KYC → 未入金 → 未交易，逐级推进
- ✅ 每个 Push 只含 **1 个 CTA**，不堆砌多重目标
- ❌ 避免在无行为状态下连续 Push 3 天以上（产生骚扰感）

**激活期（7-30 天）— 习惯养成：**
- ✅ Push 频率降至 **每周 2-3 次**，内容从「催促」转向「引导探索」
- ✅ 结合行情波动触发实时 Push（用户持仓币种异动）
- ✅ App 内 Banner + 推荐卡片承担主要曝光，减少系统级 Push

**沉睡期（14-90 天）— 召回节点：**
- ✅ **14 天** 节点：首次 Push 召回 + 小额奖励（如交易手续费减免）
- ✅ **30 天** 节点：Push + 邮件 + 弹窗三通道叠加，加大奖励力度
- ✅ **60 天** 节点：最后一轮大额回归激励（如返现券 + 空投）
- ✅ **90 天+**：降频至每月 1 次，避免浪费资源

## 六、行业参考：MEXC 抹茶的精细化弹窗模型

搜索过程中找到的 MEXC 弹窗策略可作为行业参考标杆：

| 策略维度 | 具体做法 |
|---------|---------|
| **弹窗分类** | 拉新注册 → 活动推广 → 用户召回 → 信息传递 → 转化引导（KYC/VIP） |
| **触发时机** | 首次访问（黄金 3 秒）、行为触发（浏览币种＞30 秒）、退出挽留 |
| **用户分层** | 按新老用户、资产等级、行为偏好差异化弹窗 |
| **设计原则** | 文案简洁、CTA 明确、必须有关闭按钮、「弹窗是服务不是打扰」 |
| **数据支撑** | A/B 测试 + AI 算法实时调优，ROI 提升 40%+ |

## 七、关键发现与建议关注

1. **ExchangeEpsilon 的 Widget 个性化首页是趋势** — 用 AI 推荐替代人工弹窗，把选择权还给用户
2. **ExchangeZeta 的代理商内容 Push（3 次/天）是最激进的触达手段** — 但可能有合规/用户体验风险
3. **ExchangeGamma 的 KOL 联盟军 + 高返佣模式** — App 内克制但站外凶猛，是一种差异化路径
4. **三个梯队的策略分化明显**：ExchangeEpsilon 靠产品力，ExchangeZeta 靠运营力，ExchangeGamma 靠社区力
5. **弹窗和 Push 的「度」是关键** — MEXC 的经验表明，科学设置（时机+分层+关闭选项）可以同时提升转化和体验
