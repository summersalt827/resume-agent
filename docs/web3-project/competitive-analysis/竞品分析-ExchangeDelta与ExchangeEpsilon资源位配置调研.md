# 竞品分析：ExchangeDelta 与 ExchangeEpsilon 资源位配置调研

**文档版本**：V1.0
**调研时间**：2026-05-21
**调研范围**：ExchangeDelta、ExchangeEpsilon 交易所 APP 首页资源位架构、配置规则、更新频率

## 一、ExchangeEpsilon — 资源配置体系

### 1.1 背景

ExchangeEpsilon 于 **2025年6月** 推出 "ExchangeEpsilon UI Refined" 改版，核心变化是从**固定布局 → 可定制 Widget 体系**。

### 1.2 首页架构

ExchangeEpsilon 将首页元素分为两类：

| 类型 | 管理方 | 用户操作 | 说明 |
|------|--------|----------|------|
| **Fixed Elements**（固定元素） | 运营团队统一配置 | 不可移除 | 全量用户统一展示，保证关键信息触达 |
| **Customizable Widgets**（可定制部件） | 用户自行管理 | 拖拽、增删、调整大小 | Pro Mode 下用户可按需配置 |

### 1.3 固定运营位

| 运营位 | 类型 | 更新频率 | 内容 |
|--------|------|----------|------|
| **Asset Performance Overview** | 固定（可折叠） | 实时 | 用户资产 AUM、PnL%（1D/7D/30D/180D/360D） |
| **New User Onboarding Banner** | 固定 | 版本级固定 | 新用户引导，标准化固定位置 |
| **Featured Opportunities Banner** | 轮播 | 事件驱动 | 新手任务、Launchpool、Pre-Market、新币上线、Copy Trading、Simple Earn |

### 1.4 可定制 Widget 库

| Widget | 更新频率 | 数据来源 |
|--------|----------|----------|
| AI Trending | 实时 | 整合 X 与 ExchangeEpsilon Square 社交信号的热点币种 |
| Coin List | 实时 | 币种行情列表 |
| Coin Spotlight | 实时 | 用户自选币追踪 |
| Copy Trading 排行榜 | 实时 | 跟单交易员排名 |
| ETF Net Flow | 实时 | BTC/ETH ETF 资金流入流出 |
| Fear & Greed | 每日 | 市场恐惧与贪婪指数 |
| Hot Category | 24h 滚动 | 各赛道 24h 市值变化排名 |
| Simple Earn | 实时 | 理财产品 APR 排行 |

### 1.5 运营特点

- Banner 更新为**事件驱动**，与 Launchpool / 新币上线 / 活动排期自动联动
- 高峰期可能一天换多次 Banner，淡季可能数天不变
- 所有 Fixed Elements 有**降级兜底**（默认品牌引导）
- 用户**不可关闭** Fixed Elements，但 Widget 可自由增删
- **2025-2026 Roadmap**：持续增加新 Widget，进一步优化 AI 驱动的个性化推荐

## 二、ExchangeDelta — 资源配置体系

### 2.1 背景

ExchangeDelta 于 **2025年7月** 推出 "Welcome to a new era of ExchangeDelta" 改版，核心变化是采用**双模式系统**（Lite/Pro）。

### 2.2 首页架构（自上而下）

```
┌─────────────────────────────────┐
│  Header                          │
│  （设置 / 在线客服 / 通知 / 扫码） │
├─────────────────────────────────┤
│  资产概览（实时，快速查看）       │
├─────────────────────────────────┤
│  🔴 滚动公告 Banner（横向滑动）   │  ← 系统通知 / 维护公告 / 活动上线
│  [Banner 1] [Banner 2] [Banner 3]│     后台 CMS 动态推送
├─────────────────────────────────┤
│  热门产品卡片轮播（横向滑动）     │  ← Copy / Bots / Earn 等产品入口
│  [Copy] [Bots] [Earn] [More...]  │     点击"更多"查看全部
├─────────────────────────────────┤
│  自选交易对 + 热门/新上币种        │  ← 垂直列表 / 网格
├─────────────────────────────────┤
│  活动 / 促销列表                  │  ← 进行中的活动
├─────────────────────────────────┤
│  Bottom Tab Bar                  │
│  行情 │ 衍生品 │ 交易 │ 资产      │
└─────────────────────────────────┘
```

### 2.3 各资源位更新频率

| 资源位 | 更新频率 | 驱动方式 | 备注 |
|--------|----------|----------|------|
| **滚动公告 Banner** | 动态推送 | CMS 后台推送，按发布时间倒序 | 系统通知 / 维护 / 合约到期 / 活动上线 |
| **热门产品卡片** | 按需调整 | 运营主动配置 | 与产品迭代同步 |
| **活动/促销列表** | 按活动排期 | 定时上下线 | 与营销日历同步 |
| **新币上线公告** | 月均 1-3 次 | 上新节奏 | 典型 3 日时间线：充值 → 交易 → 提现 |
| **Launchpad/Launchpool** | 月均 1-2 次 | 项目募资排期 | 每期含预热 → 认购 → 分发 → 上线交易 |

### 2.4 改版关键变化

- **双模式系统**：Lite Mode（新手友好）/ Pro Mode（高级交易者）
- **Pro Mode 支持首页快捷入口自定义**（类似 ExchangeEpsilon Widget 但更轻量）
- **高对比度设计**，简化图标，信息层级更清晰
- 设计语言三支柱：Clean & Friendly / Youthful & Tech / Iconic by Design

### 2.5 新币上线运营节奏

每次新币上线标配 **Token Splash 活动**：
1. 新用户充值奖励（专属奖池）
2. 交易竞赛（奖池通常数百万枚新币）
3. 完整 3 日上线时间线（D1 充值开放 → D1/D2 现货交易 → D3 提现开放）

### 2.6 通知类型体系

ExchangeDelta 通知中心按 Tab 分类：

| Tab | 内容 |
|-----|------|
| System Notification | 充提币通知 |
| Latest Events | 活动通知 |
| Announcements | 平台重大公告 |
| Rewards | 任务和奖励提醒 |
| TradingView Alerts | 图表预警 |
| News | 市场资讯 |
| Strategy Signals | 交易策略信号 |

## 三、ExchangeEpsilon vs ExchangeDelta vs ExchangeAlpha 对比

| 维度 | ExchangeEpsilon | ExchangeDelta | ExchangeAlpha APP |
|------|---------|-------|--------|
| 首页资源位数量 | 3 固定 + N 可定制 | 4-5 固定运营区 | **34 个资源位** |
| Banner 更新方式 | 事件驱动轮播 | 公告推送 + CMS 配置 | 排期驱动 + 主动配置 |
| 用户自定义 | Pro Mode Widget 自由拖拽 | Pro Mode 快捷入口自定义 | 无 |
| 灰度发布 | 未公开 | 未公开 | ✅ 完整灰度体系（10%→30%→50%→100%） |
| 人群定向 | 未公开（推测有基础定向） | 未公开（推测有基础定向） | ✅ 6 大维度 50+ 预置人群包 |
| AB 测试 | 未公开 | 未公开 | ✅ AB 实验平台对接 |
| 多语言 | 全球 30+ 语言 | 全球 20+ 语言 | ✅ 支持多语言配置 |
| Push 频控 | 未公开 | 未公开 | ✅ 类型/渠道/时段三维频控 |
| 优先级体系 | 后端排序逻辑 | 按发布时间倒序 | ✅ 330-363 数值优先级 |
| 排期管理 | 未公开 | 未公开 | ✅ 固定排期/循环排期/定时上下线 |
| 数据监控 | 未公开 | 未公开 | ✅ 8 核心指标 + 告警规则 |
| 设计理念 | 少而精 + AI 个性推荐 | 双模式 + 内容优先 | 全面覆盖 + 精细化运营 |

### 核心差异

ExchangeEpsilon 和 ExchangeDelta 走的是**「少而精」**路线：
- 首页运营位控制在 **5 个以内**
- 侧重**用户个性化能力**（Pro Mode Widget）
- Banner 内容由**事件/活动自动驱动**，运营配置量小

ExchangeAlpha APP 的差异化优势：
- 资源位体系**更丰富**（34个），覆盖更多运营场景
- 运营精细度**远超竞品**（人群定向 / 灰度 / AB / 频控）
- 但缺乏**用户自定义能力**，首页灵活度不如竞品

## 四、行业 Push 推送最佳实践

### 4.1 频控基准

| 用户阶段 | 推送频率 | 策略 |
|----------|----------|------|
| 新手 Onboarding（Day 0-7） | 每天 ≤ 1 条 | 欢迎 → KYC → 首充 → 首交易 递进引导 |
| 激活期（Day 7-30） | 每周 2-4 条 | 事件触发为主（交易提醒、市场异动） |
| 留存期（Day 30+） | 每周 2-5 条 | 个性化推荐 + 周报汇总 |
| 沉默召回（7-30天未活跃） | 7-14 天 1 条 | 渐进式触达：轻度 → 中度 → 强激励 |
| 流失召回（30天+） | **总量 ≤ 8-10 条后停止** | 超过 10 条无响应即终止触达 |

### 4.2 关键规则

| 规则 | 数据 |
|------|------|
| **硬上限** | 每周 ≤ 5 条/用户（跨所有渠道累计） |
| **免打扰时段** | 21:00-09:00 不推送（交易提醒/安全通知除外） |
| **事件触发 vs 定时广播** | 事件触发转化率是定时广播的 **2 倍** |
| **行为细分 vs 泛推** | 细分推送 CTR **9.35%** vs 泛推 **0.66%**（14x 差距） |
| **跨渠道频控** | Push + 邮件 + 短信 需全局累计，避免多通道叠加 |
| **用户偏好设置** | 提供分类开关（交易/活动/资讯/安全），比二元开关减少退订率 |

### 4.3 推送优先级

| 级别 | 类型 | 频控策略 |
|------|------|----------|
| P0（关键） | 安全告警、风控通知、支付失败 | 始终送达，可突破频控和免打扰 |
| P1（重要） | 交易确认、保证金催缴、强平预警 | 实时送达，单通道 |
| P2（互动） | 活动推广、产品推荐、市场资讯 | 批量发送，受频控限制 |

### 4.4 发送时间窗口

| 时段 | 内容类型 |
|------|----------|
| 09:00-10:00 | 资产/行情早报 |
| 13:00-14:00 | 产品/功能推荐 |
| 19:00-21:00 | 活动/奖励/周报总结 |
| 21:00-09:00 | **静默**（仅 P0/P1 可突破） |

## 五、对 ExchangeAlpha APP 的建议

### 5.1 短期优化（可立即执行）

| 建议 | 说明 |
|------|------|
| **Push 频控收紧** | 当前配置支持高频推送，建议统一收至 5 条/周硬上限 |
| **Banner 事件联动** | 上新/Launchpad/活动排期自动触发对应资源位配置，减少人工操作 |
| **NEW 标识自动过期** | 参考团队 SOP 中已有的 30 天规则，增加后台自动关闭提醒 |

### 5.2 中期规划

| 建议 | 说明 |
|------|------|
| **首页做减法** | 34 个资源位中低频位可收进「更多」或 Widget 化，减少首页信息密度 |
| **增加 Pro Mode 自定义** | 让高级用户自行配置首页布局（参考 ExchangeEpsilon Widget + ExchangeDelta 快捷入口） |
| **资源位 A/B 测试常态化** | 当前已有 AB 平台对接，可对 Banner/弹窗/任务位持续跑实验 |

### 5.3 长期方向

| 建议 | 说明 |
|------|------|
| **AI 驱动的个性化资源位** | 参考 ExchangeEpsilon AI Trending，基于用户交易偏好/持仓/搜索行为动态推荐资源位内容 |
| **资源位效果归因闭环** | 从「曝光→点击→转化→GMV」全链路追踪，自动优化资源位分配 |
| **运营自动化** | 排期冲突检测、人群包非空校验、灰度数据自动达标检测 |

### 5.4 ExchangeAlpha APP 差异化优势（应保持）

- **人群定向体系**：6 维度 50+ 人群包，远超市面竞品
- **灰度发布流程**：10%→30%→50%→100% 分步放量
- **AB 测试集成**：实验平台对接，可量化运营效果
- **完整 SOP 体系**：配置规范 + 运营操作 + 版本发布三套文档
