## 二、各资源位配置字段明细

### 2.1 弹窗运营位

#### 字段配置

| 字段名称 | 字段标识 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 资源位置 | resource_position | 文本 | 是 | 固定值"弹窗运营位" |
| 优先级 | priority | 数字 | 是 | 数值越大优先级越高 |
| 备注 | remark | 文本 | 否 | 配置说明或标识 |
| 投放开始时间 | start_time | 日期时间 | 是 | 弹窗生效起始时间 |
| 投放结束时间 | end_time | 日期时间 | 是 | 弹窗生效截止时间 |
| 投放渠道 | channels | 多选 | 是 | 支持全部/指定渠道（A/B/C/D） |
| 弹窗标题 | popup_title | 文本 | 是 | 弹窗顶部标题文案 |
| 弹窗背景图 | popup_bg_image | 图片 | 是 | 建议尺寸 750×1334，支持 JPG/PNG |
| 弹窗引导文案 | popup_guide_text | 文本 | 是 | 弹窗内引导用户操作的文案 |
| 跳转按钮文案 | btn_text | 文本 | 是 | 按钮显示文字，如"去参与" |
| 跳转链接 | btn_link | 链接 | 是 | 点击按钮跳转的目标页面 |
| 展示频控 | frequency_limit | 下拉单选 | 是 | 每用户展示频率（每次/每天一次/仅首次） |
| 活动名称 | activity_name | 文本 | 是 | 关联活动名称 |
| 活动开始时间 | activity_start | 日期时间 | 是 | 活动生效起始时间 |
| 活动结束时间 | activity_end | 日期时间 | 是 | 活动生效截止时间 |
| 活动背景图 | activity_bg_image | 图片 | 是 | 活动页背景图 |
| 奖励阶段 | reward_stages | 动态列表 | 是 | 支持新增多个阶段（见下方子表） |

#### 运营控制配置

| 字段名称 | 字段标识 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 上线开关 | is_active | 开关 | 是 | 默认关闭，开启后配置生效；支持紧急下线 |
| 灰度比例 | grayscale_ratio | 数字(%) | 否 | 0-100，默认100（全量）；如30表示仅30%用户可见 |
| 人群定向 | crowd_targeting | 多选 | 否 | 关联人群包，支持新用户/老用户/会员等级/注册时间/消费金额区间等维度，详见2.5 |
| 地域定向 | region_targeting | 多选 | 否 | 按省/市/区筛选，支持黑白名单，详见2.6 |
| 设备定向 | device_type | 多选 | 否 | iOS / Android / 全平台，默认全平台 |
| AB测试分组 | ab_group | 单选 | 否 | 关联AB实验分组（A/B/C），详见2.7 |
| 关联实验ID | experiment_id | 文本 | 否 | 对接AB实验平台的实验编号 |
| 投放频次上限 | frequency_cap | 数字 | 否 | 单用户每日/每周/总计最大展示次数，0为不限 |
| 频次周期 | frequency_period | 下拉单选 | 否 | 每日 / 每周 / 活动期内总计 |

#### 奖励阶段子表字段

| 字段名称 | 字段标识 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 阶段序号 | stage_order | 数字 | 是 | 第N阶段 |
| 人数规模 | crowd_size | 下拉单选 | 是 | 如"100-200人" |
| 图片资源 | stage_image | 图片 | 是 | 该阶段奖品展示图 |
| 奖品名称 | prize_name | 文本 | 是 | 如"小玩偶" |
| 奖品价值（元） | prize_value | 数字 | 是 | 奖品单价，单位元 |
| 奖品数量 | prize_count | 数字 | 是 | 该阶段投放数量 |
| 中奖概率（‰） | win_rate | 数字 | 是 | 千分比，如 100 代表 10% |

#### 展示逻辑

| 规则 | 说明 |
|---|---|
| 触发时机 | 用户进入首页时触发弹窗请求 |
| 优先级判断 | 同时段存在多条弹窗配置时，展示 priority 最高的一条 |
| 时间校验 | 仅当当前时间在 start_time ~ end_time 范围内时展示 |
| 渠道校验 | 当前用户渠道必须在配置的 channels 列表中 |
| 频控逻辑 | 每次：每次进入都展示；每天一次：同一天内仅首次进入展示；仅首次：用户生命周期内仅展示一次 |
| 活动时间校验 | 弹窗投放时间与活动时间取交集，仅交集时段内有效展示 |
| 弹窗层级 | 弹窗优先级高于所有非弹窗资源位，展示时覆盖在页面最顶层 |
| 关闭行为 | 用户点击关闭按钮或蒙层后弹窗消失，下次触发时按频控规则决定是否再次展示 |

#### 异常处理

| 异常场景 | 处理方式 |
|---|---|
| 背景图加载失败 | 展示默认占位背景图，弹窗功能不受影响 |
| 按钮跳转链接无效 | 按钮置灰不可点击，点击后 Toast 提示"活动暂不可用" |
| 投放时间与活动时间错位 | 后台校验拦截，保存时提示"投放时间与活动时间不匹配，请检查" |
| 奖励阶段概率总和异常 | 概率总和超过 1000‰ 时保存警告，不强制拦截但标注风险 |
| 奖品库存为零 | 弹窗正常展示，按钮跳转至活动页后由活动页处理库存不足逻辑 |
| 网络请求超时 | 静默跳过本次弹窗，不阻塞首页其他内容加载 |

#### 多语言配置

| 字段 | 多语言支持 | 说明 |
|---|---|---|
| 弹窗标题 | 支持 | 按语言包配置不同语言文案 |
| 弹窗引导文案 | 支持 | 按语言包配置不同语言文案 |
| 跳转按钮文案 | 支持 | 按语言包配置不同语言文案 |
| 奖品名称 | 支持 | 按语言包配置不同语言文案 |
| 弹窗背景图 | 不支持 | 图片内嵌文字需按语言版本分别上传 |
| 活动背景图 | 不支持 | 图片内嵌文字需按语言版本分别上传 |
| 奖励阶段图片 | 不支持 | 图片内嵌文字需按语言版本分别上传 |

### 2.2 首页跑马灯

#### 字段配置

| 字段名称 | 字段标识 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 资源位置 | resource_position | 文本 | 是 | 固定值"首页跑马灯" |
| 跑马灯文案 | marquee_text | 文本 | 是 | 滚动展示的文字内容，支持变量替换（如用户名、奖品名） |
| 文本颜色 | text_color | 颜色选择 | 否 | 默认白色，可自定义 |
| 跳转链接 | link | 链接 | 否 | 点击跳转目标页，可为空则不可点击 |
| 投放开始时间 | start_time | 日期时间 | 是 | 跑马灯展示起始时间 |
| 投放结束时间 | end_time | 日期时间 | 是 | 跑马灯展示截止时间 |
| 投放渠道 | channels | 多选 | 是 | 支持全部/指定渠道 |
| 展示状态 | status | 单选 | 是 | 启用 / 停用 |

#### 运营控制配置

| 字段名称 | 字段标识 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 上线开关 | is_active | 开关 | 是 | 默认关闭，开启后配置生效；支持紧急下线 |
| 灰度比例 | grayscale_ratio | 数字(%) | 否 | 0-100，默认100（全量） |
| 人群定向 | crowd_targeting | 多选 | 否 | 关联人群包，详见2.5 |
| 地域定向 | region_targeting | 多选 | 否 | 按省/市筛选，详见2.6 |
| 设备定向 | device_type | 多选 | 否 | iOS / Android / 全平台，默认全平台 |
| AB测试分组 | ab_group | 单选 | 否 | 关联AB实验分组，详见2.7 |
| 关联实验ID | experiment_id | 文本 | 否 | 对接AB实验平台的实验编号 |

#### 展示逻辑

| 规则 | 说明 |
|---|---|
| 展示位置 | 固定在首页顶部通知栏区域，位于顶部运营头图下方 |
| 滚动方式 | 文案从右向左匀速滚动，循环播放 |
| 滚动速度 | 默认 4 秒滚动一轮，文案过长时按字数等比延长 |
| 多条文案 | 多条跑马灯文案启用时，按创建时间倒序依次轮播，间隔 1 秒 |
| 点击行为 | 配置跳转链接时，点击跑马灯区域跳转对应页面；未配置链接时不可点击 |
| 时间校验 | 仅当当前时间在 start_time ~ end_time 范围内且 status 为"启用"时展示 |
| 渠道校验 | 当前用户渠道必须在配置的 channels 列表中 |
| 变量替换 | 文案中支持 {username}、{prize_name}、{amount} 等变量，运行时替换为实际值 |

#### 异常处理

| 异常场景 | 处理方式 |
|---|---|
| 文案为空 | 跑马灯区域隐藏，不占位，首页布局自动上移 |
| 变量替换失败 | 未匹配的变量以原文显示，不阻断其他变量替换 |
| 跳转链接无效 | 点击后无反应，不跳转，不弹提示（避免干扰浏览） |
| 多条文案均过期 | 跑马灯区域自动隐藏 |
| 网络请求超时 | 使用上次缓存的跑马灯数据展示，缓存有效期 30 分钟 |

#### 多语言配置

| 字段 | 多语言支持 | 说明 |
|---|---|---|
| 跑马灯文案 | 支持 | 按语言包配置不同语言文案，变量标识不变 |
| 文本颜色 | 不涉及 | 颜色值无语言差异 |

| 所有配置过期 | 自动回退展示默认品牌头图 |
| 网络请求超时 | 使用本地缓存图片展示，缓存有效期 24 小时 |

将 Banner 资源位（弹窗运营位、顶部运营头图、首页跑马灯）与平台运营事件自动联动，减少人工配置操作，提升响应速度。

#### 联动事件类型

| 事件类型 | 触发条件 | 联动资源位 | 自动操作 |
|---|---|---|---|
| 新币上线 | 后台新币状态变更为"即将上线" | 顶部运营头图（351）、行情区资源位（330）、搜索热词 | 自动创建 Banner 配置草稿，预填素材模板，运营审核后上线 |

| 交易大赛开始 | 活动状态变更为"进行中" | 弹窗运营位（352）、顶部运营头图（351）、PUSH推送 | 自动创建弹窗配置，关联活动 ID，一键审核上线 |
| 新功能上线 | 版本状态变更为"已发布" | 弹窗运营位（352）、导航菜单 NEW 标识 | 自动开启 NEW 标识，同步创建引导弹窗草稿 |

### 2.8-2 排期冲突自动检测

#### 检测规则

| 冲突类型 | 检测时机 | 处理方式 |
|---|---|---|
| 同资源位时间重叠 | 配置保存时实时检测 | **强制拦截**，提示冲突排期详情，列出冲突配置 ID 和时间段 |

### 2.22 PUSH推送

#### 推送优先级体系（P0-P2 三级）

> **参考来源**：对标行业 Push 推送最佳实践（ExchangeEpsilon / ExchangeDelta / Pushwoosh 2025 Fintech Benchmarks），建立 P0-P2 三级推送优先级体系。

| 优先级 | 级别说明 | 频控策略 | 免打扰突破 | 典型场景 |
|---|---|---|---|---|
| **P0（关键）** | 安全/风控/支付类，必须触达用户 | 可突破所有频控限制 | 可突破 | 安全告警、风控通知、支付失败、账户异常登录 |
| **P1（重要）** | 交易/保证金/清算类，实时性强 | 单通道发送，受全局频控（日≤5条） | 可突破 | 交易确认、保证金催缴、强平预警、订单成交 |
| **P2（互动）** | 活动/推荐/资讯类，批量发送 | 批量发送，受类型+全局双重频控限制 | 不可突破 | 活动推广、产品推荐、市场资讯、任务提醒 |

**P0/P1/P2 与推送类型映射**：

| 推送类型 | 对应优先级 | 说明 |
|---|---|---|
| 系统通知 | P0 | 安全告警、风控通知、支付失败等始终送达 |
| 交易提醒 | P1 | 价格预警、订单成交、止盈止损触发，实时送达 |
| 活动推送 | P2 | 新活动上线、交易大赛、空投福利，批量发送受限频控 |
| 个性化推荐 | P2 | 热门币种推荐、理财推荐、跟单推荐 |
| 召回推送 | P2 | 流失用户召回、沉默用户激活 |
| 任务提醒 | P2 | 新人任务未完成提醒、成长任务进度 |

#### 推送类型分类

| 推送类型 | 类型标识 | 典型场景 | 触发方式 | 优先级 |
|---|---|---|---|---|
| 系统通知 | system | 账户安全、系统维护、合规公告 | 事件触发 / 定时推送 | P0 |
| 交易提醒 | trade_alert | 价格预警、订单成交、止盈止损触发 | 事件触发（实时） | P1 |
| 活动推送 | activity | 新活动上线、交易大赛、空投福利 | 定时推送 / 手动触发 | P2 |

#### Push 用户生命周期频控基准

> **参考来源**：对标行业 Push 推送最佳实践，按用户生命周期阶段差异化频控策略。

| 用户阶段 | 推送频率 | 策略 |
|---|---|---|
| 新手 Onboarding（Day 0-7） | 每天 ≤ 1 条 | 欢迎 → KYC → 首充 → 首交易 递进引导 |
| 激活期（Day 7-30） | 每周 2-4 条 | 事件触发为主（交易提醒、市场异动） |
| 留存期（Day 30+） | 每周 2-5 条 | 个性化推荐 + 周报汇总 |
| 沉默召回（7-30天未活跃） | 7-14 天 1 条 | 渐进式触达：轻度 → 中度 → 强激励 |
| 流失召回（30天+） | **总量 ≤ 8-10 条后停止** | 超过 10 条无响应即终止触达 |

#### 展示逻辑

| 规则 | 说明 |
|---|---|
| 频控执行顺序 | P0/P1 优先级判断 → 推送类型独立频控 → 跨渠道合并频控 → 免打扰时段检查 → 全局日/周/月频控 → 通过则发送 |
| 免打扰逻辑 | 免打扰时段（21:00-09:00）：P0 始终发送，P1 可突破发送，P2 延迟至次日 09:00 后发送 |
| 优先级穿透 | P0（关键）可突破所有频控限制；P1（重要）可突破免打扰但受全局周频控限制；P2（互动）完全受频控和免打扰限制 |
| 跨渠道合并频控 | Push + 邮件 + 短信 全局累计计数，同一用户多通道触达不叠加超限 |
| 分批投递 | 开启 batch_delivery 时，按 batch_ratio 分批发送，每批间隔5分钟，单批不超过总用户10%；P0 推送不分批 |
| 多渠道组合 | 同一推送配置多个渠道时，各渠道独立发送、合并计频控（跨渠道合并模式） |
| 变量替换 | push_body 支持 {username}、{prize_name}、{amount} 等变量，运行时替换为实际值 |
| 人群定向校验 | 发送前实时校验用户是否命中人群包，人群包为空时**阻断发送** |
| 渠道降级 | APP推送失败时（用户关闭推送权限），自动降级为站内信触达，计入跨渠道频控 |
| 用户偏好过滤 | 发送前校验用户推送分类偏好开关，用户关闭对应分类时跳过该条推送（P0 除外） |
| 召回上限 | 流失用户（30天+）累计召回推送超过 10 条且无响应，自动终止触达 |

#### 异常处理

| 异常场景 | 处理方式 |
|---|---|
| 推送服务不可用 | 自动重试3次（间隔1/5/15分钟），仍失败则记录日志并触发告警 |
| 推送量超出预期 | 开启分批投递，每批间隔5分钟；未开启时记录日志并告警 |
| 用户已关闭APP推送权限 | 跳过APP推送，降级为站内信；两者均不可达时记录日志 |
| DeepLink跳转失败 | 跳转至APP首页，不弹错误提示 |
| 频控服务异常 | 降级为不发送（宁可少发不误发），触发告警通知运营人员 |
| 推送内容审核未通过 | 阻断发送，后台状态标记为"审核不通过"，通知运营人员修改 |
| 目标人群包为空 | 阻断发送，后台标记"人群包为空"，不发送 |
| 变量替换失败 | 未匹配的变量以原文显示，不阻断发送 |

| 规则 | 说明 |
|---|---|

| 默认态 | 搜索框展示默认占位文案或轮播热词，左侧搜索图标，右侧可展示扫一扫/相机图标 |

| 人群定向 | 配置了人群定向的热词配置，仅对命中人群的用户生效；未命中则展示默认占位文案 |
| 渠道校验 | 当前用户渠道必须在配置的设备定向/地域定向范围内 |

| 灰度实验效果 | grayscale_lift | 实验组CTR / 对照组CTR - 1 | 评估灰度实验对菜单效果的影响 |

#### 告警规则

| 告警场景 | 触发条件 | 告警级别 | 通知方式 |

| 某菜单项曝光为0但配置为上线 | 持续超过2小时 | 中 | 企业微信 |
| 下拉展开率异常 | 较前7日均值下降超50% | 中 | 企业微信 |
| NEW标识效果不达预期 | 展示3天后NEW标识CTR提升<5% | 低 | 邮件 |
