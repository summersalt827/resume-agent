## 二、需求接收规范

### 2.1 需求接收标准

合约业务方发起资源位需求时，必须提供以下信息：

**必填信息**：
- [ ] 活动/功能名称
- [ ] 推广目标（拉新/促活/转化）
- [ ] 目标人群（如：合约高频用户、新用户、VIP用户）
- [ ] 投放时间（开始-结束）
- [ ] 跳转链接（必须是合规的DeepLink）
- [ ] 素材（图片/文案，需法务审核）
- [ ] 关联活动ID

**可选信息**：
- [ ] 指定资源位偏好
- [ ] 特殊人群定向（如：仅印度用户）
- [ ] AB测试需求

### 2.2 需求评估时效

| 需求类型 | 评估时长 | 说明 |
|----------|----------|------|
| 标准配置（已有模板） | 2小时内 | 四宫图、搜索热词、跑马灯等 |
| 中等配置（需人群定向） | 4-6小时 | 头图、Banner、任务类 |
| 复杂配置（弹窗+PUSH组合） | 1-2个工作日 | 弹窗+多渠道PUSH联动 |
| 新功能配置（无模板） | 2-3个工作日 | 需产品确认方案 |

## 三、配置执行规范

### 3.1 合约弹窗配置标准流程

```
接收需求 → 人群包选择 → 频控设计 → 配置起草 → 审核 → 测试 → 上线
```

**Step 1：人群包选择**

| 推广目标 | 推荐人群包 | 定向逻辑 |
|----------|-----------|----------|
| 合约功能拉新 | 现货高频交易用户 AND 新注册用户（0-7天） | 取交集（AND） |
| 合约促活 | 合约低频交易用户 AND 活跃交易用户 | 取交集（AND） |
| 合约高净值用户运营 | 高额资产用户（≥10,000 USDT）AND 合约交易用户 | 取交集（AND） |
| 合约全量推广 | 合约交易用户 OR 现货高频交易用户 | 取并集（OR） |

**Step 2：频控设计**

| 推广场景 | 频控设置 | 说明 |
|----------|----------|------|
| 新功能上线 | 每天一次 | 防止重复打扰 |
| 限时活动 | 仅首次 | 活动期内仅展示一次 |
| 持续性推广 | 每周一次 | 保持持续曝光 |

**Step 3：配置起草**

必须填写字段：
- [ ] 资源位置：弹窗运营位
- [ ] 优先级：352（如有冲突加急协调）
- [ ] 投放时间：与活动保持一致
- [ ] 人群定向：确认目标人群包
- [ ] 频控设置：每次/每天一次/仅首次
- [ ] 跳转链接：确认DeepLink可用

### 3.2 合约PUSH配置标准流程

```
接收需求 → 推送类型确认 → 频控配置 → 内容审核 → 发送排期 → 执行发送
```

**Step 1：推送类型确认**

| 合约业务场景 | 推送类型 | 优先级 | 频控限制 |
|-------------|----------|--------|----------|
| 合约强平预警/保证金催缴 | 交易提醒 | P1（重要） | 实时送达，单通道 |
| 合约交易价格预警 | 交易提醒 | P1（重要） | 实时送达，单通道 |
| 合约大赛开赛通知 | 活动推送 | P2（互动） | 每日≤2条，受全局周频控 |
| 合约跟单功能推荐 | 个性化推荐 | P2（互动） | 每日≤1条，受全局周频控 |
| 合约功能使用提醒 | 任务提醒 | P2（互动） | 每日≤1条，受全局周频控 |
| 合约流失用户召回 | 召回推送 | P2（互动） | 每周≤2条；30天+流失≤10条后停止 |

**Step 2：频控配置**

合约业务 PUSH 频控需额外注意（对标行业 Push 推送最佳实践）：
- 全局周硬上限：**同一用户每周 ≤5 条**（跨渠道累计）
- 合约用户交易频繁，避免 PUSH 过度打扰导致卸载
- 优先使用合约专属人群包（合约高频/合约跟单用户），细分推送 CTR 可达泛推的 14 倍
- **发送时间窗口**：
  - 09:00-10:00：行情早报/合约市场异动
  - 13:00-14:00：合约产品推荐/跟单推荐
  - 19:00-21:00：合约大赛/活动/策略周报
  - 21:00-09:00：**静默时段**，仅 P0（安全通知）/ P1（强平预警/保证金催缴）可突破
- **免打扰时段**：默认 21:00-09:00 不推送 P2 级别内容
- 用户关闭"交易提醒"分类开关时，合约价格预警不发送（P1 强平预警除外）
- 流失用户（30天+）累计召回推送超过 10 条且无响应，自动终止触达

**Step 3：内容审核**

合约业务PUSH内容必须通过法务审核：
- [ ] 无虚假宣传（"保证盈利"等）
- [ ] 无过度承诺（"最高收益"等）
- [ ] 风险提示合规（合约交易有风险等）
- [ ] DeepLink跳转链路正确

### 3.3 合约事件驱动 Banner 联动

> **V1.1 新增**：对标 ExchangeEpsilon 事件驱动 Banner 机制，合约业务重大事件可自动联动资源位配置。

| 合约事件 | 触发条件 | 自动联动资源位 | 运营操作 |
|---|---|---|---|
| 合约跟单新功能上线 | 功能版本发布 | 弹窗运营位（352）+ 四宫图（350）+ PUSH | 系统自动创建配置草稿，运营审核素材后一键上线 |
| 合约交易大赛 | 活动状态变更为"进行中" | 弹窗运营位（352）+ 跑马灯 + PUSH + 搜索热词 | 关联活动 ID，自动排期同步 |
| 合约强平预警（自动） | 风控系统触发 | PUSH 推送（P1-重要） | 全自动，无需运营操作 |

### 3.4 合约AB测试配置

当合约业务方需要对不同素材/文案做AB测试时：

1. **测试目标确认**：
   - CTR（点击率）→ 优化素材
   - CVR（转化率）→ 优化落地页
   - 打开率（PUSH）→ 优化标题/文案

2. **AB实验配置**：
   - 关联实验ID
   - 分组方案（对照组A / 实验组B / 实验组C）
   - 流量比例（建议 30%/30%/40%）
   - 核心指标（选择1-2个）

3. **实验周期**：
   - 最小样本量：每组≥1000曝光
   - 统计显著性：P<0.05（95%置信度）
   - 建议周期：3-7天

## 四、排期与上线规范

### 4.1 合约资源位排期规则

| 资源位 | 排期规则 | 冲突处理 |
|--------|----------|----------|
| 弹窗运营位（352） | 同时段仅一条，按优先级排序 | 高优活动优先，联系运营协调 |
| 顶部运营头图（351） | 同时段仅一条，按优先级排序 | 提前3天预约 |
| 底部运营四宫图（350） | 同时段可多条，优先级排序 | 提前2天预约 |
| PUSH推送 | 按频控独立发送，无互斥 | 注意日频控上限 |
| 首页跑马灯 | 可多条轮播，无互斥 | 提前1天预约 |

### 4.2 合约业务优先预约通道

合约业务重要活动可申请**优先预约通道**：

**申请条件**（满足其一）：
- 合约交易大赛（参与人数≥10000）
- 合约重大功能上线（产品定义为P0）
- 合规要求的紧急通知

**预约流程**：
1. 合约业务方提交预约申请（提前5个工作日）
2. 运营评估资源位可用性
3. 确认优先级和排期
4. 锁定资源位

## 五、数据反馈规范

### 5.1 合约资源位数据反馈模板

每次合约资源位上线后，运营需在 **T+1（T为上线日）** 提供数据反馈：

```
【合约资源位数据反馈】

一、配置信息
- 配置名称：[名称]
- 资源位：[位置]
- 投放时间：[时间]
- 人群定向：[人群包]
- 关联活动ID：[ID]

二、数据概览
| 指标 | 数值 | 说明 |
|------|------|------|
| 曝光量 | XX |  |
| 点击量 | XX |  |
| CTR | X.X% |  |
| 转化数 | XX |  |
| CVR | X.X% |  |

三、数据对比
| 指标 | 本次 | 上期均值 | 变化 |
|------|------|----------|------|
| CTR | X.X% | X.X% | +XX% / -XX% |
| CVR | X.X% | X.X% | +XX% / -XX% |

四、结论与建议
- 本次效果：[好/一般/差，原因分析]
- 下次优化建议：[素材/人群/时段等]

五、关联活动效果
- 活动参与人数：XX
- 活动转化率：XX%
- 活动ROI：XX
```

### 5.2 合约PUSH数据追踪

| 指标 | 定义 | 关注重点 |
|------|------|----------|
| 送达率 | 送达数/发送数 | <80%需排查 |
| 打开率 | 打开数/送达数 | <15%需优化标题 |
| 转化率 | 完成目标行为数/打开数 | 与落地页体验相关 |
| 卸载率 | 推送后24h卸载率 | >5%触发告警 |

**PUSH数据异常处理**：
- 送达率异常下降 → 检查推送服务状态
- 打开率下降30%+ → 建议优化推送标题/内容
- 卸载率>5% → 立即暂停该批次推送，排查原因

## 六、问题与升级

### 6.1 常见问题处理

| 问题类型 | 处理方式 | 升级条件 |
|----------|----------|----------|
| 资源位被占用 | 协调优先级或调整排期 | 协调失败升级负责人 |
| 人群包命中人数为0 | 检查定向条件 | 无法解决升级产品 |
| 跳转链接失效 | 紧急下线 → 通知业务方 | 技术问题升级技术负责人 |
| 素材审核不通过 | 联系法务确认原因 | 合规问题升级合规负责人 |

### 6.2 升级流程

```
业务方反馈问题 → 运营评估（30分钟内） → 能解决立即处理
                                    ↓
                            无法解决
                                    ↓
                        升级负责人（1小时内）
                                    ↓
                        仍无法解决
                                    ↓
                        升级产品/技术（4小时内）
```

### 6.3 服务时效承诺

| 问题类型 | 响应时间 | 解决时间 |
|----------|----------|----------|
| 紧急下线 | 立即响应 | 5分钟内 |
| 配置修改 | 30分钟内 | 2小时内 |
| 数据查询 | 1小时内 | T+1提供 |
| 新需求评估 | 2小时内 | 1-3个工作日 |
