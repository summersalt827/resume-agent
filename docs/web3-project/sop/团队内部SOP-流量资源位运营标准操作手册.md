## 一、资源位配置标准流程

### 1.1 新建资源位配置流程

```
接收需求 → 资源位选型 → 配置起草 → 人群定向确认 → 排期规划 → 配置审核 → 测试预览 → 正式上线 → 数据监控
```

| 步骤 | 操作人 | 操作内容 | 时长 |
|------|--------|----------|------|
| 1.接收需求 | 运营/业务方 | 接收业务方资源位需求，明确投放目标 | — |
| 2.资源位选型 | 运营 | 根据需求类型选择合适资源位（见1.2资源位选型指南） | 0.5小时 |
| 3.配置起草 | 运营 | 在后台填写资源位配置字段 | 1-2小时 |
| 4.人群定向确认 | 运营 | 确认目标人群包（见三、人群定向使用规范） | 0.5小时 |
| 5.排期规划 | 运营 | 规划投放起止时间，设置定时上下线 | 0.5小时 |
| 6.配置审核 | 负责人 | 审核配置字段、定向、排期是否符合规范 | 0.5-1小时 |
| 7.测试预览 | 运营 | 在测试环境预览配置效果 | 0.5小时 |
| 8.正式上线 | 运营 | 开启上线开关，确认生效 | — |
| 9.数据监控 | 运营 | 监控曝光/点击/CTR数据（见六、数据监控规范） | 上线后持续 |

**总时长**：正常配置约 **4-6小时**（不含排期等待时间）

## 二、配置字段填写规范

### 2.1 必填字段检查清单

每次配置保存前，必须检查以下字段：

**基础字段**：
- [ ] 资源位置：选择正确的资源位类型
- [ ] 优先级：数值越大越优先，范围330-363（弹窗类352-353最高）
- [ ] 投放开始/结束时间：确认时间范围正确
- [ ] 投放渠道：选择全部或指定渠道
- [ ] 展示状态：启用/停用

**运营控制字段**：
- [ ] 上线开关：配置完成前保持关闭
- [ ] 人群定向：确认目标人群包
- [ ] 频控设置：弹窗类必须设置频控（每次/每天一次/仅首次）
- [ ] 灰度比例：新功能建议灰度放量（30%→50%→100%）

### 2.2 优先级使用规范

```
363 限时挑战（最高）
362 专属任务
361 成长任务
360 新人专享任务
356 首次交易任务
355 注册后首充任务
353 新用户引导弹窗
352 弹窗运营位
351 顶部运营头图
350 底部运营四宫图
340 福利中心入口
331 交易对推荐位
330 行情区资源位
```

**规则**：
- 同类型多条配置，数值大的优先展示
- 优先级相同时按创建时间倒序取最新
- 弹窗类优先级高于所有非弹窗资源位

## 三、人群定向使用规范

### 3.1 标准人群包选用流程

```
明确投放目标 → 选择人群维度 → 选用预置人群包 → 确认组合逻辑 → 预览人群规模
```

**Step 1：明确投放目标**
- 新用户转化 → 用户生命周期维度
- 合约功能推广 → 交易行为维度
- 区域活动 → 区域市场维度
- VIP用户运营 → 资产等级维度

**Step 2：选择人群维度**

| 维度 | 适用场景 | 预置人群包数量 |
|------|----------|---------------|
| 用户生命周期 | 新用户引导、沉默召回、流失用户 | 10+ |
| 交易行为 | 合约/现货/跟单/网格推广 | 10+ |
| 渠道来源 | 邀请/广告/自然流量区分 | 7+ |
| 区域市场 | 俄罗斯CIS/土耳其/印度/东南亚等 | 9+ |
| 资产等级 | 零资产/小额/中额/高额/鲸鱼用户 | 7+ |
| 活跃度 | 日活高频/沉默/浏览未交易 | 8+ |

**Step 3：选用预置人群包（标准包）**

| 人群包名称 | 标识 | 适用场景 |
|-----------|------|----------|
| 新注册用户（0-3天） | new_user_3d | 新人引导弹窗、首充任务 |
| 合约高频交易用户 | futures_high_freq | 合约新功能推广、Trading Bot推荐 |
| 现货高频交易用户 | spot_high_freq | 现货新币推荐、VIP升级引导 |
| 沉默用户（15-30天未登录） | silent_15d | 召回推送、限时挑战 |
| 俄罗斯/CIS用户 | region_russia_cis | 俄语活动、合约活动 |
| 高额资产用户（≥10,000 USDT） | high_balance | VIP灵活储蓄、专属任务 |
| PUSH高响应用户 | push_responder | 重要活动优先PUSH触达 |

**Step 4：组合逻辑确认**
- 多个人群包叠加使用 → 取交集（AND）
- 多个人群包任选其一 → 取并集（OR）
- 默认使用交集（AND）逻辑

**Step 5：人群规模预览**
- 配置保存前预览预计命中人数
- 命中人数为0 → 检查定向条件，排查冲突
- 命中人数过少（<100） → 放宽定向条件

### 3.2 典型人群组合策略

| 运营目标 | 人群包组合 | 投放资源位 |
|----------|-----------|-----------|
| 合约新功能推广 | 合约高频交易用户 AND 俄罗斯CIS用户 | 弹窗运营位 + PUSH |
| 新用户首充 | 新注册用户（0-3天）AND 零资产用户 AND 土耳其用户 | 新用户引导弹窗 + 首充任务 + PUSH（土语） |
| 沉默用户召回 | 沉默用户（15-30天）AND 中额资产用户 | PUSH召回 + 限时挑战 |
| Earn产品推广 | 现货低频交易用户 AND 中额资产用户 | Banner头图 + PUSH |
| VIP升级引导 | 高额资产用户 AND 活跃交易用户 | 专属任务 + PUSH个性化推荐 |

## 四、PUSH推送操作规范

### 4.1 PUSH配置标准流程

```
新建推送 → 选择推送类型 → 填写推送内容 → 设置运营控制 → 配置触发规则 → 审核 → 发送
```

### 4.2 推送优先级体系（P0-P2 三级）

> **参考来源**：对标行业 Push 推送最佳实践（ExchangeEpsilon / ExchangeDelta / Pushwoosh 2025 Fintech Benchmarks），建立 P0-P2 三级推送优先级体系。行为细分推送 CTR 9.35% vs 泛推 0.66%（14x 差距），事件触发转化率是定时广播的 2 倍。

#### 优先级定义

| 优先级 | 级别说明 | 频控策略 | 免打扰突破 | 典型场景 |
|---|---|---|---|---|
| **P0（关键）** | 安全/风控/支付类 | 可突破所有频控限制 | 可突破 | 安全告警、风控通知、支付失败、账户异常登录 |
| **P1（重要）** | 交易/保证金/清算类 | 单通道发送，受全局周频控（≤5条） | 可突破 | 交易确认、保证金催缴、强平预警、订单成交 |
| **P2（互动）** | 活动/推荐/资讯类 | 批量发送，受类型+全局双重频控 | 不可突破 | 活动推广、产品推荐、市场资讯、任务提醒 |

#### 推送类型与优先级映射

| 推送类型 | 优先级 | 频控限制 | 适用场景 |
|----------|--------|----------|----------|
| 系统通知 | P0 | 始终送达，无频控限制 | 安全通知、系统维护、合规公告 |
| 交易提醒 | P1 | 实时送达，单通道 | 价格预警、订单成交、止盈止损触发 |
| 活动推送 | P2 | 每日≤2条（受全局周频控） | 新活动、大赛 |
| 个性化推荐 | P2 | 每日≤1条（受全局周频控） | 理财推荐、跟单推荐 |
| 任务提醒 | P2 | 每日≤1条（受全局周频控） | 任务进度提醒 |
| 召回推送 | P2 | 每周≤2条；30天+流失用户总量≤10条后停止 | 沉默召回、流失召回 |

### 4.3 频控配置规范

**每次发送前必须确认**：
- [ ] 全局周硬上限：**同一用户每周 ≤5 条**（跨 Push + 邮件 + 短信累计）
- [ ] 单日上限：默认 2 条（P2 活动类 ≤2 条，P2 推荐类 ≤1 条）
- [ ] 跨渠道合并频控：**默认开启**，Push + 邮件 + 短信全局累计，避免多通道叠加
- [ ] 免打扰时段：**21:00-09:00 不推送**（P0 始终送达，P1 可突破，P2 延迟至次日 09:00 后）
- [ ] 推送时段：按内容类型匹配发送窗口
- [ ] 优先推送高响应人群（PUSH高响应用户优先触达）
- [ ] 用户偏好过滤：发送前校验用户推送分类开关，用户关闭对应分类时跳过（P0 除外）

#### 发送时间窗口规范

| 时段 | 内容类型 | 说明 |
|---|---|---|
| 09:00-10:00 | 资产/行情早报 | 用户起床后首屏触达，打开率最高窗口 |
| 13:00-14:00 | 产品/功能推荐 | 午休时段，用户浏览意愿强 |
| 19:00-21:00 | 活动/奖励/周报总结 | 晚间休闲时段，活动参与意愿高 |
| 21:00-09:00 | **静默时段**（仅 P0/P1 可突破） | P2 推送延迟至次日 09:00 后发送 |

#### 用户生命周期 Push 策略

| 用户阶段 | 推送频率 | 策略 |
|---|---|---|
| 新手 Onboarding（Day 0-7） | 每天 ≤ 1 条 | 欢迎 → KYC → 首充 → 首交易 递进引导 |
| 激活期（Day 7-30） | 每周 2-4 条 | 事件触发为主（交易提醒、市场异动） |
| 留存期（Day 30+） | 每周 2-5 条 | 个性化推荐 + 周报汇总 |
| 沉默召回（7-30天未活跃） | 7-14 天 1 条 | 渐进式触达：轻度 → 中度 → 强激励 |
| 流失召回（30天+） | **总量 ≤ 8-10 条后停止** | 超过 10 条无响应即终止触达 |

#### 用户推送偏好设置

运营人员应引导用户管理推送分类偏好，而非使用二元开关：
- [ ] 在 PUSH 设置页提供分类开关：**交易提醒 / 活动通知 / 市场资讯 / 安全通知**
- [ ] 用户关闭某分类后，该分类推送不发送（P0 安全通知除外）
- [ ] 定期分析退订率较高的分类，优化内容质量
- [ ] 分类开关比二元开关降低退订率约 30-50%

**频控执行顺序**：
```
P0/P1 优先级判断 → 推送类型独立频控 → 跨渠道合并频控 → 免打扰时段检查 → 用户偏好过滤 → 全局周频控（≤5条） → 发送
```

### 4.4 推送内容规范

| 字段 | 规范 | 示例 |
|------|------|------|
| 推送标题 | 限30字符，吸引眼球 | "BTC突破10万U！" |
| 推送内容 | 限80字符，含行动号召 | "立即参与瓜分50,000 USDT奖励 →" |
| 跳转链接 | 必须配置，点击跳转目标页 | deeplink://contract_copy_trading |
| 关联活动ID | 必须填写，便于效果归因 | ACT_2025_0501 |

**禁止出现**：
- 绝对化用语（"最高"、"第一"、"保证"）
- 过度营销（"立即充值10000U"等）
- 未经法务审核的合规文案

### 4.5 事件驱动 Banner 联动

> **参考来源**：对标 ExchangeEpsilon Banner 事件驱动轮播机制，Banner 更新与 Launchpool / 新币上线 / 活动排期自动联动，减少人工配置。

#### 支持联动的事件类型

| 事件类型 | 自动联动资源位 | 运营操作 |
|---|---|---|
| 新币上线 | 顶部运营头图（351）、行情区资源位（330）、搜索热词 | 后台自动创建配置草稿，运营审核素材后一键上线 |
| Launchpad/Launchpool 开启 | 弹窗运营位（352）、首页跑马灯、PUSH 推送 | 自动排期 + 自动推送，运营确认文案 |
| 交易大赛开始 | 弹窗运营位（352）、顶部运营头图（351）、PUSH 推送 | 关联活动 ID，自动同步活动排期 |

#### 运营操作流程

```
平台上新事件发生 → 后台自动创建资源位配置草稿 → 运营审核素材/文案 → 一键上线 → 事件结束后自动下线
```

### 4.6 异常情况处理

| 异常场景 | 处理方式 |
|----------|----------|
| 推送服务不可用 | 自动重试3次（间隔1/5/15分钟），仍失败告警 |
| 推送量超出预期 | 分批投递，每批10%，间隔5分钟；P0推送不分批 |
| 用户关闭推送权限 | 降级为站内信触达，计入跨渠道频控 |
| DeepLink跳转失败 | 跳转至APP首页 |
| 频控服务异常 | 降级为不发送（宁可少发不误发），触发告警 |
| 人群包为空 | **阻断发送**，标记"人群包为空"，通知运营人员 |
| 跨渠道频控超限 | 自动跳过，取优先级最高的推送发送 |
| 用户召回上限触发 | 累计10条无响应，自动终止该用户召回触达 |

### 5.2 NEW标识使用规范

NEW标识（浅黄绿色标签）用于新上线功能，吸引用户关注。

**NEW标识申请条件**：
- 功能上线时间 ≤ 30天
- 功能已完成测试验收
- 有配套运营推广计划

**NEW标识使用流程**：
1. 运营申请 → 负责人审批 → 配置开启NEW标识
2. 功能上线30天后 → 运营主动关闭NEW标识
3. 持续展示超30天 → 负责人提醒关闭
4. **V1.1 新增**：后台自动监控 NEW 标识持续时长，超 30 天自动触发邮件+企业微信提醒；超 45 天自动关闭 NEW 标识并通知负责人

## 六、数据监控规范

### 6.1 日常监控指标

**资源位核心指标**：

| 指标 | 正常范围 | 异常告警 |
|------|----------|----------|
| 曝光量 | >1000/天 | 曝光为0且配置上线 |
| 点击率(CTR) | 3-15% | CTR < 1% 或 CTR突降30% |
| 转化率(CVR) | 1-10% | CVR < 0.5% |
| 跳出率 | <50% | 跳出率 > 80% |

**PUSH核心指标**：

| 指标 | 正常范围 | 异常告警 |
|------|----------|----------|
| 送达率 | >90% | 送达率 < 80%（P0 告警） |
| 打开率 | 20-40% | 打开率 < 10%（P1 告警） |
| 卸载率 | <1% | 卸载率 > 5%（P0 告警） |
| 退订率 | <0.5%/周 | 退订率突升 50%（P0 告警） |
| P0 推送延迟 | <1 分钟 | 延迟 >1 分钟（P0 告警） |
| 跨渠道超限 | 0 次 | 单用户周超 5 条（P1 告警） |

### 6.2 告警处理流程

```
触发告警 → 评估影响范围 → 定位原因 → 采取行动 → 记录复盘
```

| 告警级别 | 处理时效 | 处理方式 |
|----------|----------|----------|
| P0（红色-关键） | 立即响应，5 分钟内定位，15 分钟内解决 | 一键下线 → 排查原因 → 修复后重新上线 → 24h 内复盘 |

## 七、紧急情况处理

### 7.1 紧急下线流程

当发生以下情况时，立即执行紧急下线：

- 配置错误导致用户大量反馈
- 跳转链接失效
- 业务突发风险事件
- 图片/内容出现合规问题

**紧急下线操作**：
1. 关闭资源位上线开关（1分钟内生效）
2. 在群里通知相关业务方
3. 定位问题原因
4. 修复后重新上线（或取消本次配置）
