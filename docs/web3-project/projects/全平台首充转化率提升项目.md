# 全平台注册未充值用户首充转化率提升

---

## STAR 总览

| 维度 | 内容 |
|------|------|
| **S**ituation | 全平台存量注册用户中约 60% 完成 KYC 但未首充，首充转化率长期横盘在 12%-15%，各区域差异大，缺乏体系化的激活策略 |
| **T**ask | 6 个月内将全局首充转化率从 14% 提升至 22%，建立可持续的激活引擎 |
| **A**ction | 按人货场三维重构激活链路 — 用户分层 × 激励产品匹配 × 场景触达节奏 |
| **R**esult | 首充转化率 14% → 24%，超额完成；首充用户 30 日交易留存 61%；单用户激活 CAC 下降 18% |

## 一、Situation — 背景与现状

### 业务现状

> 注: ExchangeAlpha APP 中 KYC 为非必选项, 用户可跳过 KYC 直接充值交易。因此漏斗为双路径并行。

```
全平台用户漏斗:

  注册用户 (100%)
    │
    ├─ 已首充 (15%)  ──────────────────────────────┐
    │   ├─ 已 KYC (9%)                              │
    │   └─ 未 KYC (6%)  ← 充了钱但不认证, 提现受限  │
    │                                               │
    └─ 未首充 (85%)  ← **核心问题池**               │
        ├─ 已 KYC, 未首充 (60%) ← 过了认证但没充    │
        └─ 未 KYC, 未首充 (25%) ← 什么都没做        │
                                                     │
  关键洞察:                                          │
  - KYC 不是首充的前置条件, 6% 的用户跳过 KYC 直接充值
  - 这批用户交易意愿强但缺少认证动力, 是「低 hanging fruit」
  - 60% 已 KYC 未首充 — KYC 通过了但充值环节卡住, 核心问题是支付摩擦或激励不足
  - 25% 双无用户 — 需要最低摩擦的首次行为触发
```

### 关键问题诊断

| 问题 | 数据表现 | 根因 |
|------|---------|------|
| 首充转化长期横盘 | 14% ± 2%, 连续 3 季度无增长 | 缺乏体系化激活策略, 用户注册后无有效引导 |
| 区域差异大 | 东南亚 18% vs 拉美 7% | 支付通道覆盖不均 + 统一激励无本地化 |
| 已 KYC 用户充值卡顿 | 完成 KYC 后 48h 内 70% 流失 | 没有从「通过审核」到「开始充值」的衔接动作 |
| 已充值未 KYC 用户被忽略 | 6% 存量用户充了钱但未认证, 零触达 | 激活策略假设 KYC 为必选项, 漏掉了这批高意向用户 |
| 存量沉睡用户无人触达 | 120 万未首充用户, 仅 3% 被推送到过 | 缺乏生命周期触达机制 |

| 维度 | 当前值 | 目标值 | 策略杠杆 |

## 三、Action — 人货场三维重构

### 3.1 人: 用户分层

#### 三层九类分群模型

```
维度 1 — 注册来源
  ├─ 自然流量 (Organic)
  ├─ 付费投放 (Paid)
  └─ 推荐码/KOL (Referral)

维度 2 — 行为阶段
  ├─ 新注册 (< 7 天)
  ├─ 近期沉默 (7-30 天)
  └─ 深度沉睡 (> 30 天)

维度 3 — 潜力分层 (基于注册时 IP 地区 + 设备 + 行为信号)
  ├─ 高潜: 设备为旗舰机 + 地区 ARPU 高 + 注册后有浏览交易页
  ├─ 中潜: 标准设备 + 中等 ARPU 地区 + 浏览过产品页
  └─ 低潜: 低端设备 + 低 ARPU 地区 + 注册后无后续行为
```

##### A1 — 高潜 × 新注册 <7d (最高优先级)

```
这类用户是整个激活漏斗的「皇冠区」— 本身质量高 + 处于决策黄金窗口。
策略目标不是「省成本」, 是「不惜代价让他完成首充」。

  触发时机:
    T+0 (注册后 30min): 未 KYC → Push「认证即领 $15 体验金, 限 24h」
    T+0 (KYC 通过后 5min): Push「$15 已到账 + 首充 $100 再领 $45 现金, 限 14 天」
    T+1 (未首充): Push — 个性化钩子 (见下方「行为钩子」)
    T+3 (未首充): 邮件「充值教程 + 倒计时提醒」+ 个性化交易对推荐
    T+5 (未首充): Push「最后 9 天, 错过不再有」
    T+7 (未首充): 邮件 + 站内信「你的专属奖励即将失效」

  行为钩子 (浏览过交易页 / 行情页的高潜用户):
    高潜用户中约 60% 在注册后有浏览行为, 这些行为信号是最好的钩子素材。

    ┌─────────────────────┬──────────────────────────────────────────┐
    │      行为信号       │                 Push 钩子                │
    ├─────────────────────┼──────────────────────────────────────────┤
    │ 浏览过 BTC/ETH      │ 「BTC 现价 $XX,XXX — 充值 $100 入手     │
    │ 行情页              │   你的第一笔 BTC, 再领 $45 现金」        │
    ├─────────────────────┼──────────────────────────────────────────┤
    │ 浏览过某山寨币      │ 「你关注的 $XXX 24h 涨了 X% — 充值      │
    │ (如 PEPE/WIF)       │   $100 即可交易, 首笔再送 $30 交易券」   │
    ├─────────────────────┼──────────────────────────────────────────┤
    │ 浏览过多个交易对    │ 「BTC、ETH、SOL… 你关注的都在涨 —       │
    │ (≥3 个)             │   充 $100 一把入手, 首充返 $45」         │
    ├─────────────────────┼──────────────────────────────────────────┤
    │ 浏览过盘前专区      │ 「$NEW_TOKEN 盘前价格 $X.XX — 比别人    │
    │                      │   早一步入场, 充值即享 $45 首充奖励」   │
    ├─────────────────────┼──────────────────────────────────────────┤
    │ 浏览过合约页面      │ 「用 $40 合约体验金零风险试一把 —       │
    │                      │   先充 $100 领 $45, 体验金自动解锁」    │
    ├─────────────────────┼──────────────────────────────────────────┤
    │ 浏览过 RWA/矿币     │ 「RWA 专区收益率达 X% — 充值解锁        │
    │   专区              │   $45 现金 + $40 RWA 专属体验金」        │
    ├─────────────────────┼──────────────────────────────────────────┤
    │ 浏览过理财/赚币     │ 「USDT 活期年化 X% — 充 $100 先领      │
    │                      │   $45 现金, 再赚利息, 双重收益」        │
    ├─────────────────────┼──────────────────────────────────────────┤
    │ 无浏览行为          │ 通用钩子: 「$45 首充奖励限时倒计时       │
    │   (fallback)        │   还剩 X 天, 热门交易对等你来」          │
    └─────────────────────┴──────────────────────────────────────────┘

  行为钩子实现要点:
    - 浏览数据来自埋点: page_view 事件, 取最近 3 次浏览的 symbol
    - 优先推送用户停留时间最长的交易对 (≥10s 计入)
    - 山寨币钩子必须附带实时涨跌幅 (如「24h +12.3%」), 用数据驱动冲动决策
    - 无浏览行为的用户 fallback 到平台级「今日涨幅 TOP3」通用推荐

  激励配置 (基准 ×1.5):
    L0  KYC:         15 USDT 体验金 (vs 基准 10)
    L1  首充 ≥$100:   45 USDT 现金 (vs 基准 30), 锁 7 天
    L2  首笔交易:     30 USDT 交易券 (vs 基准 20)
    L3  累计 ≥$5k:    50 USDT 现金 (同基准, 已自驱)
    L4  合约/理财:    40 USDT 体验金 (同基准)
    累计上限: 180 USDT

  渠道侧重:
    Push: 高频 (24h 间隔)
    邮件: 中频 (72h 间隔)
    站内信: 每次奖励到账即时
    不设静默时段例外: 高潜用户在 T+0 的 30min 出击不受夜间限制
```

##### A2 — 高潜 × 近期沉默 7-30d

```
过了 7 天黄金窗口但仍属高潜 — 可能是「观望型」用户, 需要一次强钩子打破犹豫。

  触发时机:
    进入该分群即时: Push「回归专属 — 首充 $100 享 $38 现金返现, 限 72h」
    倒计时 24h 剩余: Push「最后 1 天, 你的专属奖励即将作废」
    过期后: 降级为 B3 处理 (不再享受高潜专属激励)

  激励配置:
    L0  KYC (如未完成): 10 USDT 体验金
    L1  首充 ≥$100:     38 USDT 现金 (略高于基准, 但低于 A1), 锁 7 天
    L2-L4:              同基准

  文案关键:
    - 必须出现「专属」二字 (制造稀缺感)
    - 必须出现倒计时 (72h 紧迫窗口)
    - 不出现「你还没充值」的责备语气
```

##### A3 — 高潜 × 深度沉睡 >30d

```
最高 LTV 用户沉睡 — 不差钱, 差的是「在这个平台交易的理由」。

  触发时机:
    M+1 (沉睡第 1 个月): 邮件「VIP 专属回归礼遇」+ 客服 1v1 企微/WhatsApp
    M+2: 客服二次触达 (不作为主推, 避免骚扰)
    M+3+: 降级为 B3

  激励配置:
    L0  KYC (如未完成): 10 USDT 体验金
    L1  首充 ≥$100:     30 USDT 现金 + VIP 费率体验 30 天
                        (加码不体现在金额, 而是权益)
    L2-L4:              同基准

  渠道:
    客服 1v1 为主, 邮件为辅
    Push 在该阶段关闭 (用户已卸载 App 概率高)
    客服话术重点: 不是「回来吧有奖励」, 而是「最近有一批新币上线, 
                 盘前价格不错, 要不要看看」
```

##### B1 — 中潜 × 新注册 <7d (主力批量)

```
量最大的群体 — 标准化但完整的引导链路, 不需要加码但绝不漏掉。

  触发时机:
    同 A1 的 T+0 / T+1 / T+3 / T+7 节奏
    但不设夜间例外, 严格遵循 22:00-08:00 静默

  激励配置: 完全按基准配置, 无加码
    L0: 10 / L1: 30 / L2: 20 / L3: 50 / L4: 40
    累计上限: 150 USDT

  核心原则: 
    - 覆盖到位比激励额度更重要
    - 中潜用户的转化瓶颈往往是「不知道该怎么充」,
      而非「激励不够大」→ 文案侧重充值教程
```

##### B2 — 中潜 × 近期沉默 7-30d

```
中等质量用户的召回 — 通用型方案, 效率优先。

  触发时机:
    进入分群后 D+1: Push「你还有未领取的新手奖励」
    进入分群后 D+5: 邮件「充值教程 + 热门交易对推荐」
    进入分群后 D+10: Push (最后触达, 之后降级 B3)

  激励配置: 基准配置, 无加码
    文案强调「你的 $30 首充奖励还在, 注册后 14 天有效」
```

##### B3 — 中潜 × 深度沉睡 >30d

```
低成本批量唤醒 — 邮件为主, 不占用 Push 频次配额。

  触发时机:
    每月 1 次邮件, 连续 3 个月
    配合站内信 (登录后可见, 不主动 Push)

  激励配置:
    L1 首充 ≥$50:  15 USDT 现金 (降门槛到 $50, 降金额到 $15)
    L2-L4:         同基准
    目的: 低门槛先进来, 后续再追 L3/L4
```

##### C1 — 低潜 × 新注册 <7d

```
低 LTV 预期用户 — 给体验金不给现金, 零现金风险。

  触发时机:
    T+0 (注册后 1h): 站内信「完成 KYC 领 $5 体验金」
    T+2 (未 KYC): 站内信二次提醒
    不发 Push, 不浪费推送配额

  激励配置:
    L0  KYC:         5 USDT 体验金 (vs 基准 10)
    L1  首充 ≥$50:   10 USDT 体验金 (不给现金)
    L2  首笔交易:    10 USDT 交易券
    L3/L4:           不主动推送, 达标后自动发放

  累计上限: 25 USDT (全部非现金, 平台零现金风险)
```

##### C2 — 低潜 × 近期沉默 7-30d

```
仅在平台级大活动时附带触达, 不做单独运营。

  触发时机:
    仅当平台有节日活动 / 新币空投 / 交易大赛时
    站内信附带「你还有未领取的体验金」

  不设独立预算, 不设独立 Push
```

##### C3 — 低潜 × 深度沉睡 >30d

```
不投入。避免在几乎不可能转化的用户身上浪费触达配额和激励预算。
除非用户因产品自身吸引力自然回流, 否则不主动触达。
```

##### 分群运营的关键决策规则

```
规则 1 — 分群升级: 
  用户完成行为后即时重新分群
  例: C1 用户完成首充 $200 → 升级为 B1 并补发差额激励

规则 2 — 分群降级:
  高潜用户 7 天未首充 → A1 降为 A2
  高潜用户 30 天未首充 → A2 降为 A3
  中潜用户 30 天未首充 → B2 降为 B3
  低潜用户不降级 (已是底)

规则 3 — 跨群补偿:
  用户在低层级群时获得较低激励, 升级后补发差额
  例: B3 → B1 升级后, L1 差额 $15 (30-15) 以体验金形式补发
  不在用户升级时惩罚他当初的选择

规则 4 — A/B 实验分流:
  每个分群保留 10% 用户作为对照组 (不发放激励)
  用于持续测量激励的增量效果
  对照组用户达标后自动补发激励 (避免损害用户体验)
```

#### 区域 ARPU 计算口径

ARPU (Average Revenue Per User) 是区域档位划分的核心依据。建议用双口径并行。

**口径 1 — 交易 ARPU (反映活跃度, 起步首选)**

```
区域交易 ARPU = 该区域月度总手续费 / 该区域月活用户数
```

**口径 2 — 综合 ARPU (反映全生命周期价值)**

```
区域综合 ARPU = (月度总手续费 + 资金费率收入 + 理财收入 + 其他收入) / 月活用户数
```

**建议起步用口径 1**，数据最干净、最快跑通。

**Step 1 — 取 90 天数据窗口 (消除单月波动)**

```sql
SELECT
    user_id,
    region,              -- 按 KYC 国籍 或 注册 IP 归属
    SUM(trading_fee_usdt)        AS total_fee_90d,
    SUM(funding_rate_usdt)       AS total_funding_90d,
    COUNT(DISTINCT trade_date)   AS active_days_90d,
    CASE
        WHEN MAX(cumulative_deposit) = 0 OR MAX(cumulative_deposit) IS NULL THEN '未充值'
        WHEN MAX(cumulative_deposit) BETWEEN 1 AND 500   THEN '$1-$500'
        WHEN MAX(cumulative_deposit) BETWEEN 501 AND 2000 THEN '$500-$2000'
        ELSE '>$2000'
    END AS deposit_tier,
    CASE
        WHEN DATEDIFF(CURRENT_DATE, registration_date) <= 30  THEN '新用户(<30d)'
        WHEN DATEDIFF(CURRENT_DATE, registration_date) <= 90  THEN '成长(30-90d)'
        ELSE '成熟(>90d)'
    END AS lifecycle_stage
FROM user_trades_90d
GROUP BY user_id, region;
```

**Step 2 — 按月聚合**

```sql
SELECT
    region,
    deposit_tier,
    lifecycle_stage,
    DATE_TRUNC('month', trade_date) AS month,
    COUNT(DISTINCT user_id)         AS mau,
    SUM(trading_fee_usdt)           AS total_fee,
    SUM(trading_fee_usdt) / COUNT(DISTINCT user_id) AS arpu
FROM user_monthly_stats
GROUP BY region, deposit_tier, lifecycle_stage, month;
```

**Step 3 — 3 个月加权平滑**

```
区域 ARPU = (M1_ARPU × M1_MAU + M2_ARPU × M2_MAU + M3_ARPU × M3_MAU)
           / (M1_MAU + M2_MAU + M3_MAU)
```

**ARPU × 档位决策矩阵**

| 区域 ARPU (月) | 建议档位 | 激励力度 |
|:---:|:---:|------|
| ≥ $30 | A 档 | 加码补贴, 冲规模 |
| $15-$30 | B 档 | 基准配置 |
| $5-$15 | C 档 | 降档减半 |
| < $5 | D 档 | 零现金, 仅权益 |

**档位 review 规则**: 每季度用 ARPU 数据校验一次。ARPU 连续 2 个季度低于当前档位门槛 → 降档; 连续 2 个季度高于上一档门槛 → 升档。

**无精确数据时的代理指标 (快速评估)**

| 代理指标 | 计算方式 | 与 ARPU 相关性 |
|---------|---------|:---:|
| 人均首充金额 | 区域首充总额 / 首充人数 | 高 |
| 人均持仓余额 | 区域总余额 / 活跃用户数 | 高 |
| 合约参与率 | 合约活跃用户 / 总活跃用户 | 中高 |
| VIP 用户占比 | VIP 用户数 / 总活跃用户 | 中 |
| 出入金频率 | 月人均充提次数 | 中 |

**快速打分公式 (无精确数据时)**

```
区域潜力分 = (人均首充金额/全平台均值 × 0.4)
           + (合约参与率/全平台均值 × 0.3)
           + (VIP占比/全平台均值 × 0.2)
           + (日活留存率/全平台均值 × 0.1)

评分 > 1.2  → 高 ARPU 区域 (对应 A 档)
评分 0.8-1.2 → 中 ARPU 区域 (对应 B 档)
评分 < 0.8  → 低 ARPU 区域 (对应 C/D 档)
```

### 3.4 关键实验与迭代

#### 实验 1: 首充奖励面额测试 (第 1-2 周)

| 组别 | 激励 | 首充转化率 | 单人 CAC |
|------|------|-----------|----------|
| 对照 | 无 | 14.1% | — |
| $20 现金 | 充值 $100+ 返 $20 | 18.3% | $24.10 |
| $30 现金 | 充值 $100+ 返 $30 | **21.7%** | $31.50 |
| $50 现金 | 充值 $100+ 返 $50 | 22.1% | $47.80 |

**结论**: $30 是性价比最优解, $50 组的增量不值得 CAC 翻倍。后续统一采用 $30。

#### 实验 2: 锁定期对留存影响 (第 3-4 周)

| 组别 | 锁定期 | 首充转化率 | 30 日交易留存 | 7 日内提现率 |
|------|--------|-----------|-------------|-------------|
| 无锁 | 0 天 | 22.5% | 38% | 28% |
| 3 天锁 | 3 天 | 21.9% | 47% | 14% |
| 7 天锁 | 7 天 | 21.2% | **61%** | 6% |

**结论**: 7 天锁定期轻微压低首充率 (-1.3pp), 但留存提升 23pp 远超损失。采用 7 天锁。

#### 实验 3: 存量沉睡用户召回 (第 5-8 周)

| 组别 | 策略 | 触达人数 | 召回率 | 人均成本 |
|------|------|---------|--------|---------|
| 对照 | 常规推送 | 100,000 | 3.1% | — |
| 分层低门槛 | 低潜给 $10 体验金 / 高潜给 $25 现金 | 100,000 | 7.8% | $2.10 |
| 分层 + 限时 | 上述激励 + 72h 倒计时 | 100,000 | **11.3%** | $2.80 |

**结论**: 分层 + 限时组合召回率提升 8.2pp。对存量 120 万用户分批执行, 预计可回收 13.5 万首充用户。

#### 实验 4: 支付通道引导优化 (第 6-10 周)

| 优化 | 改动 | 效果 |
|------|------|------|
| 充值页展示本地化支付方式 | 按 KYC 国籍展示当地最常用支付方式前三位 | 进入充值页到发起充值转化 +18% |
| 充值教程视频 | 每个 TOP 支付方式配 30s 教学视频 | 充值成功率 +11% |
| 最小充值额下调 | 部分市场最小充值额从 $50 降到 $20 | 新兴市场首充率 +6% |

### 核心指标

```
                    实验前      目标       实际
  ─────────────────────────────────────────────

  新用户 7 日首充率  18%        30%        33%  ✓ 超额
```

### 核心指标口径说明

#### 首充用户 30 日交易留存 (45% → 61%, +16pp)

```
分子: 首充后 30 天内至少完成 1 笔交易的用户数
分母: 在统计周期内完成首充的全部用户数
口径: 仅统计首充日距统计日 ≥ 30 天的用户 (保证观察窗口完整)

  示例 (项目上线后第 3 个月数据):
    当月首充用户总量: 12,400 ← 分母
    其中在首充后 30 天内至少交易 1 次: 7,564 ← 分子
    30 日交易留存 = 7,564 / 12,400 = 61.0%

同比: 去年同期同口径留存率为 43% (当时无体系化激活, 首充后自然留存)
环比: 项目上线前一个季度为 45%, 上线后第一个完整季度 58%, 第二个季度 61%
      季度趋势: 45% → 58% → 61%, 锁定期策略和交易券引导持续生效
```

#### 单用户激活 CAC 下降 18% ($42 → $34)

```
激活 CAC = 总投入成本 / 首充用户数 (单位: $/人)

分子 (总投入成本):
  ├─ 激励发放成本: 体验金实际使用成本 + 现金返现支出 + 交易券抵扣金额
  ├─ 触达成本: Push/邮件/站内信几乎为零, 人工客服 1v1 触达分摊
  └─ 系统成本: 激活引擎开发及运维分摊 (一次性已摊完, 后续边际为零)

分母 (首充用户数): 统计周期内首次完成充值 ≥ $50 的用户数

  实验前基线 (项目启动前 3 个月均值):
    月均激励支出: $126,000
    月均首充用户: 3,000
    CAC = $126,000 / 3,000 = $42.0

  项目上线后 (稳定期第 3 个月):
    月均激励支出: $180,200 (激励总额增加, 因为覆盖了存量唤醒)
    月均首充用户: 5,300 (首充人数大幅上升)
    CAC = $180,200 / 5,300 = $34.0

  下降幅度: ($42.0 - $34.0) / $42.0 = 19.0% ≈ 18% (取整)

  下降驱动力: 分层策略让激励不再「大水漫灌」— 低潜给体验金 (零现金成本),
             高潜给现金但转化率更高 → 总成本增速 < 首充用户增速

同比: 去年同期 CAC 约 $44, 当时无分层机制, 统一发 $30 现金导致低潜用户补贴浪费
环比: 42 → 38 → 35 → 34 (逐月下降, 分层模型持续优化)
```

### 效果归因

| 策略杠杆 | 贡献度 | 说明 |
|----------|--------|------|
| 用户分层 + 差异化激励 | ~35% | 高潜用户拿到匹配的激励力度, 低潜不浪费预算 |
| 事件驱动即时触达 | ~25% | KYC 通过后 5min 内触达 → 首充率是延迟触达的 3.2 倍 |
| 7 天锁定期 + 交易引导 | ~20% | 减少「充完就走」, 留存和 LTV 显著提升 |
| 存量唤醒分批执行 | ~12% | 120 万存量用户滚动激活 |
| 支付通道本地化 | ~8% | 降低首充摩擦 |

### 长期影响

- 激活引擎从一次性项目转化为自动化系统, 后续无需人力干预
- 分层数据和 A/B 结论复用到合约激活、理财转化等后续链路
- 防薅体系完善后激励滥用率 < 0.3%
