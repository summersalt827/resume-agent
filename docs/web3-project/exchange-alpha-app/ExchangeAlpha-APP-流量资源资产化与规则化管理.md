---

| 属性 | 内容 |
|------|------|

## 二、项目背景与问题诊断

### 2.1 业务背景

ExchangeAlpha APP 作为加密货币交易平台，首页及核心页面承载着大量流量入口资源——弹窗、跑马灯、Banner头图、功能入口、四宫图、任务体系等，同时通过PUSH推送触达用户，这些资源位和触达通道直接影响用户的转化路径和平台的运营效率。

### 2.2 核心痛点

在项目启动前，平台流量资源管理存在以下关键问题：

| 痛点 | 具体表现 | 影响 |
|------|----------|------|
| 资源分散管理 | 34个资源位+PUSH推送+搜索栏+导航菜单无统一管理目录，各自独立配置 | 配置遗漏、冲突频发 |
| 配置依赖开发 | 资源位/PUSH上新修改需提需求走开发排期 | 运营效率低，响应慢（3-5天） |
| 缺乏精准投放 | 无法按人群、地域、设备定向投放；PUSH全量盲推 | 资源浪费，转化率低，用户被打扰 |
| 无灰度能力 | 功能上线即全量，无法小范围验证 | 大版本风险高 |
| 数据不可见 | 缺乏曝光/点击/转化数据监控 | 无法评估资源位效果，决策靠经验 |
| 无异常兜底 | 配置错误或服务异常时无降级策略 | 用户体验受损 |

---

## 三、解决方案：资源资产化管理体系

### 3.1 核心理念

**将流量资源视为"数字资产"**，通过系统化的管理方式，实现：
- **资产化**：建立资源位资产目录，让每个资源位有唯一标识、分类归属、状态追踪
- **规则化**：制定统一的配置规范和展示规则，让运营配置标准化、可预期
- **精准化**：通过人群/地域/设备/AB测试等维度，实现精细化投放
- **数据化**：建立监控体系，用数据驱动运营决策

### 3.2 体系一：资源位资产目录

将平台34个核心资源位进行分类管理，建立清晰的资产地图：

| 资源位分类 | 包含资源位 | 优先级范围 | 管理策略 |
|------------|-----------|-----------|----------|
| **弹窗类** | 弹窗运营位、新用户引导弹窗 | 352-353 | 频控管理，防止过度打扰 |
| **Banner类** | 顶部运营头图、行情区资源位、交易对推荐位 | 330-351 | 优先级排序，轮播机制 |
| **图标矩阵类** | 底部运营四宫图、福利中心入口 | 340-350 | 固定位展示，灵活配置 |
| **文字滚动类** | 首页跑马灯 | — | 多条轮播，变量替换 |
| **任务体系类** | 新人专享、成长任务、专属任务、限时挑战、首充任务、首次交易任务 | 355-363 | 用户生命周期匹配 |
| **信息聚合类** | 活动日历、卡券中心 | — | 按时间/状态聚合展示 |
| **PUSH推送类** | 活动推送、交易提醒、系统通知、个性化推荐、召回推送、任务提醒 | — | 频控+人群+场景触发 |
| **搜索入口类** | 搜索栏（含热词引导） | — | 常驻展示，多语言支持 |
| **AI功能入口类** | AI交易套件（网格交易、跟单等AI技能） | — | 按用户生命周期精准推荐 |
| **导航菜单类** | 交易/合约/工具/金融/活动（一级菜单5项）+ 邀请返佣/XT合伙人/经纪人/XT智能链/实验室/公告/XT博客/学院/上币/AI交易套件（"更多"下拉10项） | — | 一级常驻+二级按需展示 |

**资产目录价值**：
- 运营人员可一目了然查看所有资源位及其状态
- 快速定位可投放资源位，避免重复配置
- 资源位状态实时可见（在线/灰度/下线/过期）

### 3.3 体系二：统一配置规范

为所有资源位建立标准化的配置字段规范：

**基础配置字段（通用）**：

| 字段 | 作用 | 设计考量 |
|------|------|----------|
| 资源位置 | 唯一标识资源位归属 | 固定值，避免歧义 |
| 优先级 | 同类型资源位展示排序 | 数值越大越优先，范围330-363 |
| 投放时间 | 生效起止时间 | 时间段内才展示，过期自动下线 |
| 投放渠道 | 支持全部/指定渠道 | 多渠道差异化运营 |
| 展示状态 | 启用/停用 | 一键控制资源位生效状态 |

**优先级体系设计**：

```
任务体系（360-363）> 弹窗类（352-353）> Banner类（350-351）
> 功能入口（340）> 推荐位（330-331）
```

设计原则：高转化价值的资源位获得更高优先级；同类型资源位通过优先级实现互斥控制。

### 3.4 体系三：运营控制体系

建立8大运营控制能力，实现精准化投放：

| 控制能力 | 功能说明 | 应用场景 |
|----------|----------|----------|
| **上线开关** | 一键紧急下线，1分钟内生效 | 突发事件紧急下线 |
| **灰度比例** | 按百分比控制曝光用户比例（0-100%） | 新功能逐步放量验证 |
| **人群定向** | 基于用户标签、会员等级、消费行为定向 | VIP专属活动、新用户引导 |
| **地域定向** | 按省/市/区筛选，支持黑白名单 | 区域性活动、合规要求 |
| **设备定向** | iOS/Android/全平台 | 平台差异化运营 |
| **AB测试** | 分组实验，按UID哈希固定分组 | 资源位效果对比验证 |
| **频控管理** | 每日/每周/活动期频次上限 | 防止过度打扰用户 |
| **投放排期** | 固定排期/循环排期，定时上下线 | 自动化运营管理 |

**人群定向能力详情**：
- 支持多维度人群圈选：用户类型、会员等级、注册时间、登录活跃度、消费金额区间、累计下单次数、自定义标签
- 支持6大人群维度体系：用户生命周期（10+人群包）、交易行为（10+人群包）、渠道来源（7+人群包）、区域市场（9+人群包）、资产等级（7+人群包）、活跃度（8+人群包），累计预置50+标准人群包
- 人群组合策略支持：合约新功能精准推广（合约高频+地区）、新用户转化（0-3天新注册+零资产+区域）、沉默召回（中额资产+15-30天沉默）、VIP升级（高额资产+活跃）等典型场景
- 支持交集（AND）和并集（OR）逻辑组合
- 实时标签实时计算，离线标签T+1更新
- 人群包为空时降级为不展示，避免全量误投

### 3.5 体系四：展示规则引擎

设计10层校验链路，确保资源位精准展示：

```
①上线开关 → ②投放排期 → ③渠道校验 → ④设备定向 → ⑤地域定向
→ ⑥人群定向 → ⑦灰度比例 → ⑧AB测试分组 → ⑨频次控制 → ⑩优先级判断
```

**执行逻辑**：
- 任一环节不满足条件则不展示，逐层过滤
- 最终命中最高优先级配置展示给用户
- 弹窗类资源位层级最高，覆盖在页面最顶层

**异常降级策略**：

| 异常场景 | 降级处理 |
|----------|----------|
| 人群定向服务超时 | 降级为全量展示 + 触发告警 |
| AB实验平台不可用 | 降级为对照组配置 |
| 图片CDN故障 | 自动切换备用CDN / 展示占位图 |
| 全部资源位请求失败 | 展示静态默认布局 |
| 配置数据格式异常 | 忽略异常配置，使用缓存兜底 |

### 3.6 体系五：数据监控闭环

建立覆盖"曝光→点击→转化"的全链路数据监控体系：

**核心监控指标**：

| 指标 | 计算方式 | 业务意义 |
|------|----------|----------|
| 曝光量 | 资源位展示总次数 | 衡量资源位覆盖面 |
| 点击量 | 资源位被点击总次数 | 衡量资源位吸引力 |
| 点击率(CTR) | 点击量/曝光量 | 核心质量指标 |
| 转化率(CVR) | 转化数/点击量 | 衡量转化效果 |
| 人均停留时长 | 总停留/去重曝光人数 | 弹窗类适用 |
| 跳出率 | 无后续行为的比例 | 衡量内容匹配度 |

**告警机制**：
- 支持多维告警规则：指标低于/高于阈值、环比上升/下降
- 支持持续时间窗口，避免瞬时波动误报
- 多渠道通知：站内信/邮件/企业微信/飞书/短信
- 告警风暴防护：10分钟内超5次自动合并

### 3.7 体系六：PUSH推送管理体系

将PUSH推送纳入统一的资产化管理体系，实现"精准触达、规则可控、效果可评估"的推送运营。

#### 推送类型分类

| 推送类型 | 典型场景 | 触发方式 | 优先级 |
|----------|----------|----------|--------|
| **活动推送** | 新活动上线、交易大赛、空投福利 | 定时推送 / 手动触发 | 高 |
| **交易提醒** | 价格预警、订单成交、止盈止损触发 | 事件触发（实时） | 最高 |
| **系统通知** | 账户安全、系统维护、合规公告 | 事件触发 / 定时推送 | 最高 |
| **个性化推荐** | 热门币种推荐、理财推荐、跟单推荐 | 算法触发 / 定时推送 | 中 |
| **召回推送** | 流失用户召回、沉默用户激活 | 定时批量 / 人群触发 | 低 |
| **任务提醒** | 新人任务未完成提醒、成长任务进度 | 事件触发 + 延迟策略 | 中 |

#### PUSH推送配置字段规范

**基础配置字段**：

| 字段名称 | 字段标识 | 类型 | 必填 | 说明 |
|----------|----------|------|------|------|
| 推送名称 | push_name | 文本 | 是 | 推送任务标识，如"BTC突破10万U提醒" |
| 推送类型 | push_type | 下拉单选 | 是 | 活动推送/交易提醒/系统通知/个性化推荐/召回推送/任务提醒 |
| 推送标题 | push_title | 文本 | 是 | 推送通知栏标题，限30字符 |
| 推送内容 | push_body | 文本 | 是 | 推送通知栏正文，限80字符 |
| 推送图标 | push_icon | 图片 | 否 | 通知栏小图标，建议48×48 |
| 跳转链接 | deeplink | 链接 | 是 | 点击推送跳转的目标页面，支持DeepLink |
| 投放开始时间 | start_time | 日期时间 | 是 | 推送生效起始时间 |
| 投放结束时间 | end_time | 日期时间 | 是 | 推送生效截止时间 |
| 推送状态 | status | 单选 | 是 | 草稿/待发送/发送中/已完成/已取消 |

**运营控制配置**：

| 字段名称 | 字段标识 | 类型 | 必填 | 说明 |
|----------|----------|------|------|------|
| 人群定向 | crowd_targeting | 多选 | 是 | 关联人群包，支持新用户/老用户/会员等级/注册时间/消费行为等 |
| 地域定向 | region_targeting | 多选 | 否 | 按省/市/区筛选，支持黑白名单 |
| 设备定向 | device_type | 多选 | 否 | iOS / Android / 全平台 |
| 推送渠道 | push_channel | 多选 | 是 | APP推送/短信/邮件/站内信，支持多渠道组合 |
| AB测试分组 | ab_group | 单选 | 否 | 关联AB实验，测试不同推送文案/时机 |
| 灰度比例 | grayscale_ratio | 数字(%) | 否 | 0-100，默认100（全量） |
| 推送频控 | frequency_control | 下拉单选 | 是 | 每用户每日/每周/每月最大推送次数 |
| 频控上限值 | frequency_cap | 数字 | 是 | 同一用户在频控周期内最多收到的推送数 |
| 推送时段限制 | time_window | 时间区间 | 是 | 仅在指定时段内发送，如09:00-21:00 |
| 免打扰时段 | dnd_time_range | 时间区间 | 否 | 禁止推送的时段，如22:00-08:00 |
| 推送优先级 | push_priority | 单选 | 是 | 高/中/低；高优先级推送可突破频控限制 |
| 延迟策略 | delay_strategy | 下拉单选 | 否 | 立即推送/延迟N分钟/用户活跃时触发 |
| 关联活动ID | activity_id | 文本 | 否 | 关联平台活动，便于效果归因 |

#### PUSH频控规则设计

频控是PUSH管理的核心规则，直接决定用户体验和推送效果：

| 频控维度 | 规则说明 | 设计考量 |
|----------|----------|----------|
| **单日上限** | 同一用户每天最多收到N条推送（默认3条） | 防止推送疲劳，降低卸载率 |
| **单周上限** | 同一用户每周最多收到N条推送（默认10条） | 控制周频次，平衡触达与打扰 |
| **单月上限** | 同一用户每月最多收到N条推送（默认30条） | 月度总量控制 |
| **类型维度** | 不同推送类型独立频控（如活动类每日最多2条） | 防止单一类型过度推送 |
| **渠道维度** | APP推送和短信分别计频，合并上限 | 多渠道触达但总控不超标 |
| **免打扰时段** | 22:00-08:00不推送（系统通知/交易提醒除外） | 尊重用户休息时间 |
| **优先级穿透** | 高优先级推送（交易提醒/安全通知）可突破常规频控 | 关键信息必达 |

**频控执行逻辑**：

```
用户命中推送条件
  → 检查推送类型独立频控 → 超限则不发送
  → 检查渠道合并频控 → 超限则不发送
  → 检查当前时段是否在免打扰时段
    → 在免打扰时段 + 优先级=高 → 允许发送
    → 在免打扰时段 + 优先级≠高 → 延迟到免打扰结束后发送
  → 检查全局日/周/月频控 → 超限则不发送
  → 通过所有校验 → 推送发送
```

#### 推送触发规则

| 触发方式 | 适用场景 | 执行逻辑 |
|----------|----------|----------|
| **定时推送** | 活动推广、定期公告 | 按配置时间点批量发送，支持分批投递（防止瞬间高峰） |
| **事件触发** | 价格预警、订单成交、任务完成 | 监听业务事件，实时匹配用户并推送 |
| **人群触发** | 流失召回、新人引导 | 用户进入指定人群包后自动触发推送 |
| **延迟触发** | 注册未完成提醒、任务进度提醒 | 用户动作后延迟N分钟，未完成目标则触发 |
| **算法触发** | 个性化推荐 | 推荐算法输出结果后，按用户活跃时段择机推送 |

#### 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 推送服务不可用 | 自动重试3次（间隔1/5/15分钟），仍失败则记录日志并告警 |
| 推送量超出预期 | 分批投递，每批间隔5分钟，单批不超过总用户10% |
| 用户已关闭推送权限 | 跳过APP推送，降级为站内信触达 |
| DeepLink跳转失败 | 跳转至APP首页，不弹错误提示 |
| 频控服务异常 | 降级为不发送（宁可少发不误发），触发告警 |
| 推送内容审核未通过 | 阻断发送，通知运营人员修改 |
| 目标人群为空 | 阻断发送，后台标记"人群包为空" |

#### 推送数据监控指标

| 指标 | 计算方式 | 业务意义 |
|------|----------|----------|
| 推送发送量 | 成功送达设备数 | 衡量推送覆盖面 |
| 送达率 | 送达数/发送数 | 衡量推送通道质量 |
| 推送打开率 | 打开数/送达数 | 核心质量指标，衡量推送吸引力 |
| 推送转化率 | 完成目标行为数/打开数 | 衡量推送带来的实际转化 |
| 推送卸载率 | 收到推送后24h内卸载用户占比 | 衡量推送是否过度打扰 |
| 推送退订率 | 关闭推送权限的用户占比 | 衡量长期推送策略健康度 |
| 推送ROI | 推送带来的GMV/推送运营成本 | 衡量推送投入产出比 |

**推送专属告警规则**：

| 告警场景 | 触发条件 | 通知方式 |
|----------|----------|----------|
| 送达率异常下降 | 送达率较前7日均值下降超20% | 企业微信 + 邮件 |
| 打开率异常下降 | 打开率较前7日均值下降超30% | 企业微信 |
| 卸载率飙升 | 推送后24h卸载率超5% | 企业微信 + 短信 |
| 推送量异常 | 单次推送量超出预期200% | 企业微信 + 邮件 |
| 频控突破告警 | 高优先级推送突破频控次数超3次/天 | 企业微信 |

### 3.8 体系七：搜索栏配置体系

搜索栏是用户主动搜索的核心入口，同时承载搜索热词引导、运营词配置等运营能力，位于首页顶部常驻展示。

#### 核心功能设计

| 功能模块 | 功能说明 | 配置规范 |
|----------|----------|----------|
| **默认态展示** | 搜索框占位文案/运营热词轮播 | 默认文案"搜索币种、交易对、活动"，支持多语言 |
| **热词引导** | 运营配置的搜索热词展示 | 支持轮播模式（定时切换）和固定模式（展示置顶热词） |
| **搜索历史** | 用户搜索行为记录 | 默认开启，最多保存10条，30天自动过期 |
| **人群定向** | 按用户类型/区域展示不同热词 | 新用户/老用户/高净值用户可配置差异化热词 |

#### 热词配置规范

| 配置项 | 说明 |
|--------|------|
| 热词文案 | 如"BTC"、"新人福利"、"Trading Bot"，支持加粗高亮 |
| 热词颜色 | 默认主题色，可自定义颜色 |
| 点击行为 | 跳转搜索结果页（自动填充搜索词）或跳转指定链接 |
| 展示时段 | 支持配置热词的投放起止时间 |
| 人群定向 | 可针对特定人群包配置专属热词 |

#### 展示逻辑

- 搜索框左侧搜索图标+右侧扫一扫/相机图标
- 未聚焦时展示占位文案或轮播热词（默认3秒切换）
- 点击后跳转至搜索结果页，弹出键盘
- 人群定向热词仅对命中人群的用户展示

#### 异常降级

| 异常场景 | 降级处理 |
|----------|----------|
| 热词配置请求失败 | 展示默认占位文案 |
| 热词跳转链接失效 | 降级为跳转搜索结果页 |
| 全部热词过期 | 回退展示默认占位文案 |

### 3.9 体系八：导航菜单资源位

导航菜单是XT顶部导航栏的核心组成部分，包含一级菜单（5个固定项）和二级菜单（"更多"下拉10个功能入口）两个层级。

#### 导航菜单结构

| 层级 | 菜单项 | 说明 |
|------|--------|------|
| **一级菜单（顶部导航栏）** | 交易 / 合约 / 工具 / 金融 / 活动 | 常驻固定，不可关闭 |
| | 更多 | 触发"更多"下拉菜单 |
| **二级菜单（"更多"下拉-左列）** | 邀请返佣（NEW）/ XT合伙人 / 经纪人（NEW）/ XT智能链 / 实验室 | 左列5项 |
| **二级菜单（"更多"下拉-右列）** | 公告 / XT博客 / 学院 / 上币 / AI交易套件（NEW） | 右列5项 |

**NEW标识**：邀请返佣、经纪人、AI交易套件为新上线功能，菜单项旁展示浅黄绿色NEW标签。

#### 运营控制能力

| 控制能力 | 功能说明 |
|----------|----------|
| **上线开关** | 关闭后该菜单项对所有用户隐藏 |
| **灰度比例** | 按UID哈希分配展示比例（0-100%） |
| **人群定向** | 仅对命中人群的用户展示（如VIP用户专属"XT合伙人"入口） |
| **地域定向** | 按IP/手机归属地筛选 |
| **设备定向** | iOS / Android / Web / 全平台 |
| **语言定向** | 俄语 / 英语 / 土耳其语 / 中文等，按用户语言展示 |
| **会员等级定向** | 普通 / 银卡 / 金卡 / 钻石用户可见 |
| **AB测试** | 测试不同菜单配置（链接、文案、顺序等）的点击效果 |

#### 各菜单项专属配置

| 菜单项 | 专属配置 |
|--------|----------|
| **邀请返佣** | 返佣比例（如"20%返佣"）、专属邀请链接、受邀用户资格（全部/新用户/特定人群） |
| **XT合伙人** | 扶持计划名称、加入条件说明 |
| **经纪人** | 返佣比例区间、专属费率展示、加入条件 |
| **XT智能链** | 链简介、主网状态（运行中/维护中）、官方链接 |
| **实验室** | 研究方向、投资方向列表、联系方式 |
| **公告** | 最新公告标题、未读数量红点、公告类型筛选 |
| **XT博客** | 最新文章标题、文章分类、博客入口 |
| **学院** | 课程分类、最新课程、学习进度同步 |
| **上币** | 服务介绍、申请入口、上币须知 |
| **AI交易套件** | AI技能列表（网格交易/跟单等）、新用户引导、使用教程链接 |

#### 数据监控指标

| 指标 | 计算方式 | 业务意义 |
|------|----------|----------|
| 菜单曝光量 | 菜单项被展示次数 | 衡量展示覆盖率 |
| 菜单点击量 | 菜单项被点击次数 | 衡量菜单吸引力 |
| 菜单点击率(CTR) | 点击量/曝光量 | 衡量菜单设计合理性 |
| 下拉展开率 | "更多"点击次数/页面UV | 衡量"更多"菜单使用频率 |
| 二级菜单点击分布 | 各二级菜单点击量/总点击量 | 了解各功能相对热度 |
| NEW标识点击提升 | 开启NEW后CTR/基准CTR - 1 | 评估NEW标识效果 |

#### 告警机制

| 告警场景 | 触发条件 | 级别 |
|----------|----------|------|
| 菜单点击率异常下降 | 单日CTR较前7日均值下降超30% | 高 |
| 某菜单项曝光为0但配置上线 | 持续超过2小时 | 中 |
| 下拉展开率异常 | 较前7日均值下降超50% | 中 |
| NEW标识效果不达预期 | 展示3天后CTR提升<5% | 低 |

## 六、项目经验沉淀

### 6.1 方法论

**资产化思维**：将分散的流量资源抽象为可管理、可追踪、可评估的"数字资产"，建立资产目录和状态管理，让资源位不再是"黑盒"。

**规则化管理**：通过标准化配置规范和10层展示规则引擎，将运营经验固化为系统规则，降低人为错误风险，提升配置一致性和可预期性。

**数据闭环**：建立"配置→投放→监控→优化"的数据闭环，用数据驱动资源位的配置优化和运营决策，持续提升流量利用效率。
