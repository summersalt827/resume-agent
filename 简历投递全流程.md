# Resume Agent

AI 驱动的简历助手，支持匹配打分、生成、优化、定制、翻译、投递管理等功能。

![[resume-agent/website/img-hero.png]]

## 安装

```bash
cd ~/Desktop/resume-agent
pip install -r requirements.txt
cp .env.example .env
# 编辑 .env 填入你的 ANTHROPIC_API_KEY
```

## 使用

```bash
python main.py
```

选择对应编号即可：

1. **生成简历** — 交互式输入个人信息，AI 生成完整简历（自动保存 .md + .pdf）
2. **优化简历** — 上传已有简历，AI 分析并提供优化建议
3. **简历匹配打分** — 粘贴目标 JD，对所有简历版本按 6 维度打分排名，选择最佳版本后直接进入定制
4. **定制简历** — 粘贴目标 JD，AI 生成针对性定制简历（自动保存 .md + .pdf）
5. **翻译简历** — 中英文简历互译
6. **管理项目经验** — 录入/查看/删除/同步项目经验，供优化简历时参考
7. **投递与面试管理** — 全生命周期追踪投递、面试、反馈
8. **岗位抓取** — 搜索 LinkedIn / Wellfound / MyCareersFuture 岗位，AI 翻译 JD 后加入投递池

## 标准工作流

### 收到新 JD 时

0. **检查 STAR 项目经历** — 确认 `docs/` 下有对应公司的 STAR 拆解文档，如无则提醒用户准备：
   > 「建议先准备好 STAR 版项目经历（Situation / Task / Action / Result），可以从 `docs/` 下找现有文档参考格式。准备好后可以放进 NotebookLM，投递时作为面试准备知识库。」
1. **匹配打分** — 粘贴 JD → 扫描 `简历/` 下所有版本 → 按 6 维度逐项打分 → 展示排名 + 推荐最佳基础版本
2. **选择版本** — 根据匹配结果选择最优基础版本
3. **定制简历** — 基于选中版本 + JD 定制：
   - 按 `简历.md` 模板结构输出（Professional Summary / Work Experience / Education / Skills）
   - Work Experience 主岗位用 Strategy + Case 格式，尾部岗位精简为 bullet list
   - **自动保存 .md + .pdf** 到 `投递岗位&定制化简历/`，文件名 `{姓名}（{公司}）.md/.pdf`
   - PDF 排版：全部左对齐，PingFang 字体，无缩进
4. **加入投递清单** — 在 `data/applications.json` 新增记录，必填字段：公司、职位、渠道、简历版本（PDF 路径）、匹配分（含 6 维度明细）
5. **投递后更新** — 状态改为「已投递」，渠道等字段如有变更及时修正

### 岗位搜寻

4.5. **岗位抓取** — 选 8 → 选平台 → 输入关键词：
   - LinkedIn / Wellfound / MyCareersFuture 三个渠道
   - AI 自动翻译 JD 为中文 + 结构化提取
   - 表格展示后勾选 → 去重 → 批量写入 `data/applications.json`

### 定制简历完成后 → 模拟面试触发

5.5. **建立团队 OKR** — 根据 JD 和岗位要求，假设本岗位的团队 OKR：
   - 拆解到季度/月度目标（如：M1 触达规则上线 → M2 自助配置率 80% → M3 2 条业务线接入）
   - 每个目标配对应的 KPI（如事故率归零、CTR 提升 X%、成本降 Y%）
   - 面试中所有回答都映射到 OKR，展示「我不是来干活的，我是来扛目标的」
5.6. **启动模拟面试** — 简历生成和 OKR 完成后立即开始：
   - 收集面试材料（个人项目 STAR 拆解文档 + 竞品/行业调研文档 + 策略文档）
   - 按 JD 权重最高领域精准设置 2-3 道复合场景题
   - 每题打分 + 追问 + 给出参考答案和面试建议
   - Cover 高频盲区（如 EFTD 资产转化率、团队 OKR 思维）
   - 暴露弱项 → 针对性补救 → 下一版模拟面试验证
   - **不等到收到面试邀请**，在简历投递前就可以开始准备

### 收到面试邀请时

6. **同步面试记录** — 将公司从 `applications.json` 同步到 `data/interviews.json`
7. **加入日历** — 同步到 macOS 日历（日历名称「工作」），包含公司名、岗位、时间（默认 1 小时）
8. **面试准备** — 准备好常见问题的中英文回答，保存到投递记录备注中

### 每日投递结束回顾

9. **End-of-Day Review** — 每天投递结束时展示以下内容：
   - **今日投递清单**：公司名 + 岗位名 + 渠道，逐条列出
   - **累计数据**：总投递数 + 约面数 + 面试率（%）
   - **优化建议**：渠道策略（是否过度集中）、匹配分分布（有无 <5 分硬投）、简历版本覆盖率（裸投率）

### 简历生成规范

- **模板**：严格遵循 `简历.md` 结构
- **语言**：中英文分开，不混排；海外岗位用英文
- **真实性**：公司、职位、时间、教育背景必须真实，只调整描述角度和关键词
- **自动保存**：生成/定制完成后自动保存 .md + .pdf 到 `投递岗位&定制化简历/`
- **PDF 排版**：全部左对齐，无缩进，字体 PingFang，页边距 18mm
- **命名**：`{姓名}（{公司/版本}）.md` / `.pdf`
- **页数**：控制在 2 页以内

## 匹配打分标准（6 维度 / 10 分制）

| 维度 | 分值 | 评分逻辑 |
|------|------|----------|
| 行业匹配 | 0-2 | 同行业=2，可迁移相邻行业=1，完全不同=0 |
| 职能匹配 | 0-2 | 核心职责覆盖 70%+=2，30-70%=1，无法覆盖=0 |
| 硬性门槛 | 0-1 | 学历/执照/特定年限等硬要求全部满足=1，有一条不满足=0 |
| 经验年限 | 0-1 | JD 要求年限 ≤ 实际经验=1，差距 > 3年=0 |
| 语言/区域 | 0-2 | 完全匹配=2，部分匹配=1，关键语言不满足=0 |
| 技能/工具 | 0-2 | 所列技能 70%+满足=2，30-70%=1，核心技能缺失=0 |

**判定**：8-10 优先投递 | 6-7 可投 | 5 边缘 | **< 5 不投**

## 岗位抓取

### 支持平台

| 平台 | 抓取方式 | 说明 |
|------|---------|------|
| LinkedIn (领英) | DuckDuckGo 搜索 `site:linkedin.com/jobs/view` | 全球岗位，关键词搜索 |
| Wellfound (AngelList) | DuckDuckGo 搜索 `site:wellfound.com/jobs` | 初创/远程岗位，过去一周 |
| MyCareersFuture | DuckDuckGo 搜索 `site:mycareersfuture.gov.sg` | 新加坡政府平台，过去一月 |

### 流程

```
python main.py → 选 8 → 选平台 → 输入关键词
    → DuckDuckGo 搜索岗位
    → Claude 翻译 + 结构化提取
    → Rich 表格展示（公司/职位/地点/薪资/要求摘要）
    → 勾选心仪岗位 → 去重 → 批量写入投递池
```

### 数据格式

抓取的岗位自动写入 `data/applications.json`，字段与手动录入一致：company / position / location / url / salary / status / notes。notes 前缀标记来源（`Wellfound:` / `MCF:` / `LinkedIn:`）。

## 面试准备与模拟面试

### 面试流程

```
收到面试邀请
     ↓
同步到 interviews.json + macOS「工作」日历
     ↓
收集面试材料（STAR拆解 + 策略文档 + 竞品调研 + JD分析）
     ↓
模拟面试（按 JD 逐轮覆盖） → 记录得分 + 弱项 + 补救建议
     ↓
正式面试
     ↓
面试后复盘 → 记录反馈
```

### 面试材料归档

面试材料统一放在 `面试/{公司}/面试材料/`，包括：

| 文件类型 | 文件名 | 用途 |
|----------|--------|------|
| **公司调研** | `{公司}调研与触达体系阶段推演.md` | 基本面 + 竞品对比 + 6维诊断 → 面试中展示判断力 |
| **公司 JD 原文** | `JD.md` | 五大领域拆解 + 打分 → 驱动模拟面试出题 |
| **个人项目 STAR 拆解** | 从 `docs/` 复制核心项目文档 | 模拟面试出题素材 + 自我介绍数据来源 |
| **竞品/行业调研** | 如 `三站Push弹窗调研.md` | 展示行业视野，回答"你怎么看竞品"类问题 |
| **策略文档** | Push策略、SOP等 | 回答专业深度问题的知识储备 |
| **模拟面试记录** | 在 `面试/{公司}/模拟面试记录.md`（上级目录） | 逐轮 Q&A + 评分 + 弱项追踪 |

### 可给模拟面试提供的文件 Prompt

在启动模拟面试时，将以下文件内容一起发给 AI：

```
模拟面试输入材料：
1. 目标 JD — 面试/{公司}/面试材料/JD.md
2. 核心项目 STAR 拆解（1-3 份）— 根据 JD 匹配，用最相关的
3. 公司调研报告 — 面试/{公司}/面试材料/{公司}调研与触达体系阶段推演.md
4. 竞品/行业调研（如有）
5. 个人简历 PDF（定制版）
```

**常见文件组合示例**：

| JD 类型 | 推荐 STAR 文档 | 推荐调研 |

| 触达渠道/中台运营 | ExchangeAlpha流量资源化 STAR + 首充转化 STAR | 目标公司调研 + 三站Push弹窗调研 |

## 反问面试官问题库

### 好问题

**角色与团队：**
- 这个岗位是新建的还是替换？之前触达是怎么协调的？
- 团队现在规模和分工？产品、运营、开发分别多少人？
- 汇报给谁？期望3个月止血还是6个月搭完体系？

**现状与挑战：**
- 各业务线 Push 是各自直发还是已有统一规则？冲突最严重的是哪两条？
- EDM 月均成本？有没有做分层投放或区域本地化？
- 触达效果归因用什么模型？

**跨部门：**
- 推动中台化阻力最大的业务线是哪条？试过推吗？
- 产研对中台是主动还是被动？有排期预留吗？

**文化与节奏：**
- 决策靠数据驱动 AB 测试，还是靠经验直觉？
- 外语能力要求？跨区域沟通主要用什么语言？

### 一般问题

- 试用期考核标准？
- 长远规划？

### 避免问

- 薪资福利（留 HR）
- 融资/盈利（敏感）
- 「公司文化怎么样」（太虚）
- JD 已写清楚的事

> 正式面试挑 **2-3 个**，优先：资源位现状 → 最大阻力 → 决策文化。

## 模拟面试提问 Prompt

### 出题框架

按 JD 五大领域分配出题权重，每轮模拟面试前指定材料来源及其权重。

```
你是目标公司的面试官，岗位是{岗位名}。严格按照 JD 的五大职责领域出题：

1. {JD领域1}（权重 ~{x}%）
2. {JD领域2}（权重 ~{x}%）
3. {JD领域3}（权重 ~{x}%）
4. {JD领域4}（权重 ~{x}%）
5. {JD领域5}（权重 ~{x}%）

面试流程：
1. 按领域顺序逐轮出题，每轮 1 道主问题
2. 候选人回答后，10 分制打分 + 指出好的方面 + 暴露的不足
3. 根据回答漏洞进行 1-2 轮追问，同样打分
4. 追问结束后给出完整参考答案 + 面试建议
5. 计算本轮均分，进入下一轮

出题原则：
- 不要纯理论题，全部用场景和冲突包装
- 从候选人 STAR 文档中提取真实业务场景
- 追问要瞄准首答中最薄弱的部分
- 参考答案必须有可操作的具体方案，不能空泛

材料权重分配示例：
- 核心项目 STAR：~50%（对应 JD 第 1-2 条，必考）
- 转化项目 STAR：~30%（对应 JD 第 1、3 条）
- 其他相关经验：~20%（对应 JD 第 4-5 条）

记录格式（每道题）：
**Q{N}**：题目
- 首答 X/10：好的方面 + 不足
- 追问 X/10：好的方面 + 不足
**本轮均分**：X/10
**参考答案**：完整方案
**面试建议**：一句话提炼
```
